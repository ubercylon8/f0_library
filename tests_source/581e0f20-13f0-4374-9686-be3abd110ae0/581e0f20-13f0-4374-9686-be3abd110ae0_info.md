# Ransomware Encryption via BitLocker

**Category**: Ransomware / Data Encryption / Impact
**Severity**: Critical
**MITRE ATT&CK**: T1070.001, T1562.004, T1082, T1083, T1486, T1490

## Description

This multi-stage security test simulates a ransomware attack that leverages Windows BitLocker for data encryption, based on NCC Group research. Unlike traditional ransomware that uses custom encryption, BitLocker-based ransomware abuses a legitimate Windows feature, making detection more challenging.

The test executes a 3-stage attack chain:
1. **Defense Evasion**: Clear event logs and manipulate firewall rules
2. **Discovery**: Enumerate system information and BitLocker availability
3. **Impact**: Encrypt data using BitLocker and inhibit system recovery

All operations use VHD-based isolation to ensure no impact to the real system.

## Technical Details

### Attack Flow

1. **Stage 1: Defense Evasion** (T1070.001, T1562.004)
   - Creates custom event log channel "F0RT1KA-Test"
   - Writes test event to the custom channel
   - Clears the custom log channel using `wevtutil cl`
   - Creates test firewall rule "F0RT1KA-Test-Rule" using `netsh advfirewall`
   - Deletes the test firewall rule
   - Removes custom event log channel

2. **Stage 2: Discovery** (T1082, T1083)
   - Queries system information using `wmic computersystem`
   - Queries OS information using `wmic os`
   - Checks Windows edition (Pro/Enterprise required for BitLocker)
   - Verifies manage-bde availability
   - Queries BitLocker status on existing volumes
   - Enumerates logical disks using `wmic logicaldisk`
   - Creates target_volumes.txt with discovery results

3. **Stage 3: Impact** (T1486, T1490)
   - Creates 100MB VHD at `C:\F0\bitlocker_test.vhd` using diskpart
   - Initializes, partitions, and formats VHD as NTFS
   - Mounts VHD with label "F0RT1KA-TEST"
   - Creates test files on VHD (documents, configs)
   - Enables BitLocker with password: `manage-bde -on <drive> -Password "F0RT1KA-Recovery-2024!" -UsedSpaceOnly`
   - Attempts VSS shadow deletion: `vssadmin delete shadows /for=<drive> /quiet`
   - Cleanup: disables BitLocker, detaches VHD, deletes VHD file

### Key Indicators

**Process Execution**:
- `wevtutil cl F0RT1KA-Test` - Event log clearing
- `netsh advfirewall firewall add rule` - Firewall manipulation
- `wmic computersystem get` - System enumeration
- `wmic logicaldisk get` - Drive enumeration
- `diskpart` - VHD operations
- `manage-bde -on` - BitLocker encryption
- `vssadmin delete shadows` - Shadow copy deletion

**File System**:
- `C:\F0\bitlocker_test.vhd` - Temporary VHD file
- `C:\F0\target_volumes.txt` - Discovery results
- `C:\F0\stage*_output.txt` - Stage execution logs
- `C:\F0\test_execution_log.json` - Structured test results

**Registry/Event Log**:
- Custom event log channel "F0RT1KA-Test"
- BitLocker key protector registration

## Detection Opportunities

1. **Event Log Clearing** (T1070.001)
   - Monitor for `wevtutil.exe` execution with `cl` (clear) parameter
   - Security Event ID 1102 (Log cleared)
   - Suspicious pattern: any event log clearing followed by encryption activity

2. **Firewall Manipulation** (T1562.004)
   - Monitor `netsh.exe` with `advfirewall firewall add rule` or `delete rule`
   - Windows Firewall service events
   - Rapid create/delete patterns indicate testing or evasion

3. **System Enumeration** (T1082)
   - `wmic.exe` process execution with `computersystem`, `os` parameters
   - High-volume WMI queries from unexpected processes
   - System enumeration followed by encryption activity

4. **Drive Discovery** (T1083)
   - `wmic.exe` with `logicaldisk` queries
   - Volume enumeration API calls
   - Pattern: drive enumeration + BitLocker commands

5. **BitLocker Encryption** (T1486)
   - `manage-bde.exe` execution with `-on` parameter
   - BitLocker API calls from non-standard processes
   - Password-protected BitLocker (unusual - normally TPM-based)
   - Encryption initiated on non-system volumes

6. **Shadow Copy Deletion** (T1490)
   - `vssadmin.exe` with `delete shadows` command
   - Volume Shadow Copy Service manipulation
   - VSS deletion combined with encryption = high-confidence ransomware

## Expected Results

### Unprotected System (Code 101)

When the attack succeeds:
- All 3 stages complete without EDR intervention
- VHD is created and encrypted with BitLocker
- VSS deletion command executes (no shadows to delete on VHD)
- Test artifacts created in `C:\F0`
- JSON log shows all stages with status "success"
- Cleanup runs automatically

### Protected System (Enhanced Detection)

**Code 126 (Execution Prevented)**:
- EDR blocks one of the stage binaries at execution
- Common detection points:
  - Stage 1: wevtutil or netsh execution blocked
  - Stage 2: wmic enumeration detected as reconnaissance
  - Stage 3: manage-bde or vssadmin blocked

**Code 105 (Binary Quarantined)**:
- Stage binary detected and quarantined before execution
- Test stops at the blocked stage
- Earlier stages show status "success"
- Blocked stage shows status "blocked"

**Code 999 (Prerequisites Not Met)**:
- Not running as Administrator
- BitLocker not available (Windows Home edition)
- Required Windows services not running
- VHD creation failed

## Behavioral Detection

This test includes opportunities for KQL-based behavioral detection:

### Event Log Clearing Detection
```kql
DeviceProcessEvents
| where ProcessCommandLine has "wevtutil" and ProcessCommandLine has "cl"
| project TimeGenerated, DeviceName, AccountName, ProcessCommandLine
```

### BitLocker Encryption Detection
```kql
DeviceProcessEvents
| where FileName =~ "manage-bde.exe"
| where ProcessCommandLine has "-on" and ProcessCommandLine has "-Password"
| project TimeGenerated, DeviceName, AccountName, ProcessCommandLine
```

### VSS Shadow Deletion Detection
```kql
DeviceProcessEvents
| where FileName =~ "vssadmin.exe"
| where ProcessCommandLine has "delete" and ProcessCommandLine has "shadows"
| project TimeGenerated, DeviceName, AccountName, ProcessCommandLine
```

### Combined Ransomware Indicator
```kql
let SuspiciousCommands = DeviceProcessEvents
| where TimeGenerated > ago(1h)
| where (FileName =~ "manage-bde.exe" and ProcessCommandLine has "-on")
    or (FileName =~ "vssadmin.exe" and ProcessCommandLine has "delete shadows")
    or (FileName =~ "wevtutil.exe" and ProcessCommandLine has "cl")
| summarize CommandCount=count(), Commands=make_set(ProcessCommandLine) by DeviceName, AccountName;
SuspiciousCommands
| where CommandCount >= 2
| extend ThreatScore = 95
| project TimeGenerated=now(), DeviceName, AccountName, ThreatScore, Detection="BitLocker Ransomware Activity", Commands
```
