## Description

Pass-the-Hash (PTH) is a lateral movement technique where attackers use captured NTLM password hashes to authenticate to remote systems. This eliminates the need to crack the hash to obtain the plaintext password.

This test simulates PTH attack patterns to validate that security controls can detect or prevent this technique.

## Attack Simulation Details

### Phase 1: NTLM Hash Pattern Simulation
- Creates files with NTLM hash format patterns
- Simulates credential artifacts from tools like Mimikatz

### Phase 2: LSASS Access Pattern
- Attempts LsaConnectUntrusted API call
- Simulates what credential dumping tools do

### Phase 3: Network Logon Simulation
- Uses LogonUser with LOGON32_LOGON_NEW_CREDENTIALS
- This logon type (Type 9) is a key PTH indicator

### Phase 4: SMB Authentication Pattern
- Simulates Impacket-style SMB authentication
- Attempts share enumeration with credential patterns

### Phase 5: Security Event Detection
- Queries Security Event Log for PTH indicators
- Checks for Event ID 4624 Type 9 events

## Detection Requirements

### Windows Security Events
- **4624** (Type 9): NewCredentials logon - PRIMARY PTH INDICATOR
- **4624** (Type 3): Network logon from unusual source
- **4648**: Explicit credential logon
- **4672**: Special privileges assigned (admin logon)

### EDR/Sysmon Events
- Process accessing LSASS memory
- Suspicious LogonUser API calls
- Unusual NTLM authentication patterns

### Network Indicators
- NTLM authentication from unusual hosts
- Lateral authentication chains
- High-frequency authentication attempts
