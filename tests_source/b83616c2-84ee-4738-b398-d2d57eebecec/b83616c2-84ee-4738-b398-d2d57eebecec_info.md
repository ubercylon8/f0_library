## Overview
This test evaluates endpoint protection against NativeDump (Nim flavor), an advanced LSASS dumping tool that bypasses traditional detection methods by using only NTAPI functions and hand-crafting Minidump files without calling MiniDumpWriteDump.

## Technique Details
- **MITRE ATT&CK**: T1003.001 (OS Credential Dumping: LSASS Memory)
- **Tactic**: Credential Access
- **Platform**: Windows
- **Permissions Required**: Administrator/SeDebugPrivilege
- **Category**: Security Test, Credential Access

## Test Description
The test performs the following actions:
1. Dropping the NimDump binary (disguised as `library_update.exe`) to disk
2. Attempting to execute the tool to dump LSASS process memory
3. Verifying if a dump file was created
4. Cleaning up any artifacts created during the test

### Technical Details
NativeDump/NimDump uses advanced techniques to evade detection:
- Uses only NTAPI functions from ntdll.dll
- Hand-crafts Minidump files without using dbghelp!MiniDumpWriteDump
- Can bypass API hooking by remapping ntdll.dll
- Creates minimal Minidump with only necessary streams (SystemInfo, ModuleList, Memory64List)
- Uses NTOpenProcessToken and NtAdjustPrivilegeToken for privilege escalation
- Employs NtQueryVirtualMemory and NtReadVirtualMemory for memory dumping
