# SafePay Ransomware Simulation & Data Staging

## Test Information

**Test ID**: 109266e2-2310-40ea-9f63-b97e4b7fda61  
**Test Name**: SafePay Ransomware Simulation & Data Staging  
**Category**: Ransomware / Data Exfiltration  
**Severity**: Critical  
**MITRE ATT&CK**: T1486, T1560.001, T1071.001, T1490  

## Description

This test simulates the SafePay ransomware family's behavior, focusing on its data staging capabilities using WinRAR before encryption. SafePay is known for its sophisticated approach to maximizing damage by first exfiltrating sensitive data before encrypting it, providing attackers with additional leverage for extortion.

## Technical Details

### Attack Flow
1. **Initial Setup**
   - Drops WinRAR.exe to C:\F0
   - Drops PowerShell script for orchestration
   - Checks for administrator privileges

2. **Decoy File Creation**
   - Creates realistic files with sensitive-looking names
   - Includes financial reports, SSN databases, password lists
   - Mimics typical high-value corporate data

3. **Data Staging**
   - Executes WinRAR with specific parameters:
     - `-v5g`: Create 5GB volume splits
     - `-m0`: No compression (faster)
     - `-mt5`: Multi-threaded operation
     - Excludes image and PST files
   - Archives sensitive file types before encryption

4. **File Encryption**
   - Simulates encryption by Base64 encoding
   - Appends .safepay extension
   - Deletes original files

5. **Ransom Note**
   - Creates readme_safepay.txt
   - Opens in Notepad to alert user
   - Contains payment instructions

6. **C2 Simulation**
   - Simulates header pattern: C4 C3 C2 C1
   - Mimics key transmission to C2 server

### Key Indicators
- Creation of WinRAR.exe in C:\F0
- PowerShell execution with bypassed policy
- Mass file operations with .safepay extension
- Creation of exfiltration.rar archive
- Ransom note creation and display

## Detection Opportunities

1. **File System Activity**
   - Rapid creation of multiple files
   - Mass renaming to .safepay extension
   - WinRAR.exe dropped to unusual location

2. **Process Behavior**
   - PowerShell with -ExecutionPolicy Bypass
   - WinRAR.exe execution with archiving parameters
   - Notepad.exe launched with ransom note

3. **Network Indicators**
   - Simulated C2 communication pattern
   - Potential data staging for exfiltration

## Expected Results

### Unprotected System (Code 101)
- All files successfully encrypted
- Ransom note displayed
- Data archive created
- No security intervention

### Protected System
- **Code 105**: WinRAR binary quarantined on drop
- **Code 126**: PowerShell execution blocked
- File encryption prevented
- Ransomware behavior detected and stopped
