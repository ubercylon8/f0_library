# SMB Protocol Hardening Validator

## Test Information

**Test ID**: b4b50f92-f19a-4aba-a119-6f0e26d54ba5
**Test Name**: SMB Protocol Hardening Validator
**Category**: Cyber Hygiene / Configuration Validation
**Severity**: Critical
**MITRE ATT&CK**: T1021.002, T1570, T1210, T1557

## Description

This cyber hygiene test validates that the SMB (Server Message Block) protocol is properly hardened on Windows endpoints. SMB is a critical network file sharing protocol that, when improperly configured, enables devastating lateral movement attacks.

The test validates five critical SMB security configurations:

1. **SMBv1 Server Disabled**: Prevents exploitation of EternalBlue and related vulnerabilities
2. **SMBv1 Client Disabled**: Blocks SMBv1 connections to potentially malicious servers
3. **Server Signing Required**: Prevents SMB relay and man-in-the-middle attacks
4. **Client Signing Required**: Ensures integrity of SMB communications
5. **SMB Encryption Enabled**: Protects data in transit from eavesdropping

**Historical Context**: SMBv1 vulnerabilities were exploited by WannaCry (May 2017) and NotPetya (June 2017), causing over $10 billion in combined damages globally. Microsoft has recommended disabling SMBv1 since 2016.

This test performs READ-ONLY validation - it does NOT simulate attacks or modify any configurations.

## Technical Details

### Validation Flow

1. **Phase 0: Initialization**
   - Verify administrator privileges (required for registry access)
   - Initialize Schema v2.0 compliant logging

2. **Phase 1: SMBv1 Server Check**
   - Read registry: `HKLM\SYSTEM\CurrentControlSet\Services\LanmanServer\Parameters\SMB1`
   - If missing, fallback to PowerShell:
     - `Get-WindowsOptionalFeature -Online -FeatureName SMB1Protocol`
     - `Get-SmbServerConfiguration | Select EnableSMB1Protocol`
   - Compliant: value = 0, Feature = Disabled, or EnableSMB1Protocol = False

3. **Phase 2: SMBv1 Client Check**
   - Read registry: `HKLM\SYSTEM\CurrentControlSet\Services\mrxsmb10\Start`
   - Service start types: 0=Boot, 1=System, 2=Auto, 3=Manual, 4=Disabled
   - Compliant: value = 4 (Disabled) or service key not found

4. **Phase 3: Server Signing Check**
   - Read registry: `HKLM\SYSTEM\CurrentControlSet\Services\LanmanServer\Parameters\RequireSecuritySignature`
   - Compliant: value = 1

5. **Phase 4: Client Signing Check**
   - Read registry: `HKLM\SYSTEM\CurrentControlSet\Services\LanmanWorkstation\Parameters\RequireSecuritySignature`
   - Compliant: value = 1

6. **Phase 5: SMB Encryption Check**
   - Execute PowerShell: `(Get-SmbServerConfiguration).EncryptData`
   - Compliant: value = True

7. **Phase 6: Compliance Determination**
   - Calculate overall compliance (all 5 checks must pass)
   - Generate summary report with remediation guidance

### Key Indicators

Security teams monitoring for configuration validation activity may observe:

- Registry reads to `HKLM\SYSTEM\CurrentControlSet\Services\LanmanServer\Parameters`
- Registry reads to `HKLM\SYSTEM\CurrentControlSet\Services\LanmanWorkstation\Parameters`
- Registry reads to `HKLM\SYSTEM\CurrentControlSet\Services\mrxsmb10`
- PowerShell execution: `Get-SmbServerConfiguration`
- PowerShell execution: `Get-WindowsOptionalFeature`

## Detection Opportunities

1. **Registry Access Monitoring**
   - Monitor for read access to LanmanServer and LanmanWorkstation registry keys
   - Could indicate reconnaissance activity assessing SMB configuration

2. **PowerShell Logging**
   - Script block logging will capture Get-SmbServerConfiguration and Get-WindowsOptionalFeature cmdlets
   - Module logging will show SmbShare and DISM module usage

3. **Process Execution**
   - Monitoring for processes reading multiple SMB-related registry keys in sequence
   - Unusual for standard applications outside of security tools

## Expected Results

### Compliant System (Code 126)

All five SMB hardening configurations are properly set:
- SMBv1 Server = Disabled (0)
- SMBv1 Client = Disabled (Start=4)
- Server Signing = Required (1)
- Client Signing = Required (1)
- SMB Encryption = Enabled (True)

The system is hardened against SMB-based attacks including EternalBlue, SMB relay, and lateral movement.

### Non-Compliant System (Code 101)

One or more configurations are missing or incorrect:
- SMBv1 may be enabled, exposing EternalBlue vulnerabilities
- SMB signing may be optional, allowing relay attacks
- SMB encryption may be disabled, exposing data in transit
- System may be vulnerable to WannaCry/NotPetya class attacks

Remediation is required immediately for systems exposed to untrusted networks.

### Test Error (Code 999)

The test could not complete due to:
- Insufficient privileges (must run as Administrator)
- Timeout exceeded (rare for this test)
- Unexpected system error

## Performance Considerations

- **SMB Signing**: Adds approximately 15% CPU overhead due to cryptographic operations on each SMB packet
- **SMB Encryption**: Adds additional CPU overhead but provides confidentiality for all SMB traffic
- **Recommendation**: Enable both signing and encryption despite overhead; the security benefits outweigh performance costs for most environments
