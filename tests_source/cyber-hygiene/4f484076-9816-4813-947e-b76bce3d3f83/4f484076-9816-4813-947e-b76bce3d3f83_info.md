## Purpose

This test validates the security configuration of a Microsoft Entra ID (formerly Azure AD) tenant against the CISA SCuBA (Secure Cloud Business Applications) baseline for AAD. It checks 8 security domains aligned to SCuBA controls MS.AAD.1.x through MS.AAD.8.x:

1. **Legacy authentication blocking** -- Ensures outdated protocols are disabled
2. **Risk-based policies** -- Validates Identity Protection is enforcing on high-risk events
3. **MFA/strong authentication** -- Confirms phishing-resistant MFA, Authenticator context, weak method removal
4. **Centralized logging** -- Verifies diagnostic settings export audit and sign-in logs
5. **Application governance** -- Restricts user app registration and consent
6. **Password policies** -- Validates NIST 800-63B non-expiring password compliance
7. **Privileged access** -- Checks GA count, PIM usage, cloud-only accounts, approval requirements
8. **Guest access** -- Validates guest role restrictions, invitation limits, domain filtering
