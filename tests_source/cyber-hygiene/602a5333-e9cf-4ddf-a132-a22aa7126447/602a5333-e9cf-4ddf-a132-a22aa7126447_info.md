## Purpose

This test validates the identity and Active Directory security configuration of a domain-joined Windows environment against CIS Benchmark Level 1 controls. It covers both on-premises AD hardening and cloud identity (Entra ID) security, providing a comprehensive hybrid identity security assessment.

### Validator Details

1. **Privileged Account Management** (CH-CIA-001 to 005)
   - Domain Admins count limited to 5 or fewer
   - Enterprise Admins and Schema Admins groups empty
   - Protected Users security group populated with privileged accounts
   - AdminSDHolder audit for orphaned elevated permissions

2. **Service Account Security** (CH-CIA-006 to 008)
   - Group Managed Service Accounts (gMSA) deployed
   - Service account passwords rotated within 365 days
   - No unconstrained Kerberos delegation on non-DC computers

3. **Authentication Hardening** (CH-CIA-009 to 012)
   - Fine-Grained Password Policies (PSOs) configured
   - KRBTGT password rotated within 180 days
   - No users with reversible password encryption
   - Kerberos pre-authentication enforced (AS-REP Roasting prevention)

4. **AD Infrastructure** (CH-CIA-013 to 018)
   - AD Recycle Bin enabled for object recovery
   - Domain functional level at least Windows 2016
   - Stale computer and user account audit (90-day threshold)
   - LDAP signing required (LDAPServerIntegrity = 2)
   - LDAP channel binding enabled (LdapEnforceChannelBinding >= 1)

5. **Group Policy Security** (CH-CIA-019 to 021)
   - Default Domain Policy minimum password length >= 14
   - Critical GPO permissions audit (no overly permissive ACLs)
   - Advanced audit policy configured with key subcategories

6. **Entra ID Controls** (CH-CIA-022 to 028) -- Optional, requires Graph API
   - MFA enforced for all users via Conditional Access
   - MFA enforced for admin roles
   - Legacy authentication blocked
   - Privileged Identity Management (PIM) enabled
   - App registration restricted to admins
   - Security Defaults or Conditional Access active
   - Break-glass accounts (>=2 Global Admins)

## Requirements

### AD Checks (Always Required)
- Domain-joined Windows machine
- RSAT: Active Directory PowerShell module
- Network connectivity to domain controller
- Standard domain user account (no admin required for read queries)

### Entra ID Checks (Optional)
- Microsoft Graph PowerShell SDK
- Service principal with read-only permissions
- `AZURE_TENANT_ID`, `AZURE_CLIENT_ID`, `AZURE_CLIENT_SECRET` environment variables
