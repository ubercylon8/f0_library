**Category**: Cyber Hygiene / Configuration Validation
**Severity**: High

## Description

This cyber hygiene test validates that the Windows Print Spooler service is properly hardened or disabled to protect against PrintNightmare (CVE-2021-34527) and related vulnerabilities. PrintNightmare is a critical vulnerability that allows remote code execution with SYSTEM privileges through the Windows Print Spooler service.

The vulnerability was so severe that CISA issued Emergency Directive 21-04, requiring federal agencies to immediately disable the Print Spooler service or apply mitigations. This test validates compliance with those security requirements.

The test performs READ-ONLY validation of:

1. **Print Spooler Service Status**: Whether the service is disabled or not running
2. **NoWarningNoElevationOnInstall**: Prevents silent driver installation without UAC
3. **UpdatePromptSettings**: Controls whether driver updates require user interaction
4. **RestrictDriverInstallationToAdministrators**: Limits driver installation to admin accounts

This test does NOT simulate any attacks - it only validates that defensive configurations are in place.

## Technical Details

### Attack Context

PrintNightmare (CVE-2021-34527) exploits the Windows Print Spooler service to achieve:

1. **Remote Code Execution**: Attackers can remotely load malicious printer drivers
2. **SYSTEM Privileges**: Code executes as NT AUTHORITY\SYSTEM
3. **Domain Compromise**: On Domain Controllers, leads to full domain compromise

The vulnerability exists because:
- Print Spooler allows remote driver installation by default
- Point and Print features allow non-admin driver installation
- No validation of driver authenticity by default

### Validation Flow

1. **Phase 0: Initialization**
   - Verify administrator privileges (required for service and registry access)
   - Initialize Schema v2.0 compliant logging

2. **Phase 1: Spooler Service Check**
   - Query service status via `sc query Spooler`
   - Read registry `HKLM\SYSTEM\CurrentControlSet\Services\Spooler\Start`
   - Determine if service is: Disabled (Start=4), Stopped, or Running

3. **Phase 2: Point and Print Restrictions Check**
   - If service is disabled/stopped: Mark as compliant (checks informational only)
   - If service is running: All restrictions are REQUIRED
   - Check `NoWarningNoElevationOnInstall` = 0
   - Check `UpdatePromptSettings` = 0
   - Check `RestrictDriverInstallationToAdministrators` = 1

4. **Phase 3: Domain Controller Detection**
   - Check for NTDS service (Active Directory)
   - Check ProductType registry value
   - Warn if Print Spooler is running on a DC (HIGH RISK)

5. **Phase 4: Compliance Determination**
   - Calculate overall compliance based on tiered logic
   - Generate summary report with specific remediation steps

### Key Indicators

Security teams monitoring for configuration validation activity may observe:

- Service Control Manager queries (`sc query Spooler`)
- Registry reads to `HKLM\SYSTEM\CurrentControlSet\Services\Spooler`
- Registry reads to `HKLM\SOFTWARE\Policies\Microsoft\Windows NT\Printers\PointAndPrint`
- Registry reads to `HKLM\SYSTEM\CurrentControlSet\Control\ProductOptions`
- Process execution: `net session` (admin check)

## Detection Opportunities

1. **Service Query Monitoring**
   - Monitor for queries to Print Spooler service status
   - Could indicate reconnaissance activity

2. **Registry Access Monitoring**
   - Monitor for reads to Spooler service registry keys
   - Monitor for reads to Point and Print policy keys

3. **Process Monitoring**
   - Monitor for the test binary execution from `c:\F0`
   - Monitor for service control utility usage

## Expected Results

### Compliant System (Code 126)

The system meets one of these conditions:
- Print Spooler service is disabled (Start = 4) - **Best Practice**
- Print Spooler service is stopped (not running)
- Print Spooler is running with ALL Point and Print restrictions:
  - NoWarningNoElevationOnInstall = 0
  - UpdatePromptSettings = 0
  - RestrictDriverInstallationToAdministrators = 1

The system is protected against PrintNightmare exploitation.

### Non-Compliant System (Code 101)

The Print Spooler service is running WITHOUT proper Point and Print restrictions:
- Any of the three Point and Print settings are insecure
- Potential for remote code execution via PrintNightmare
- Immediate remediation required

### Test Error (Code 999)

The test could not complete due to:
- Insufficient privileges (must run as Administrator)
- Timeout exceeded (rare for this test)
- Unexpected system error

## Domain Controller Considerations

Domain Controllers are HIGH-VALUE targets. The Print Spooler service on a DC:

1. **Expands Attack Surface**: Remote exploitation leads to domain compromise
2. **Provides SYSTEM on DC**: Attackers gain domain admin capabilities
3. **No Business Need**: DCs rarely need to print

**Recommendation**: ALWAYS disable Print Spooler on Domain Controllers, regardless of organizational printing requirements.
