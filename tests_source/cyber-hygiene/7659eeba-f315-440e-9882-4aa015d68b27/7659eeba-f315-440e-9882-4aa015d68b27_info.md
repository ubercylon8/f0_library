## Purpose

This bundled test validates the identity posture of Windows endpoints by checking 5 critical identity and cloud-integration configurations in a single execution. It complements the Entra ID Tenant Security Hygiene Bundle (`4f484076`) by focusing on the **endpoint side** of identity hardening:

1. Whether the device is properly joined to Azure AD / Entra ID
2. Whether Windows Hello for Business replaces password-based authentication
3. Whether the device is enrolled in MDM (Intune) for compliance enforcement
4. Whether cloud credentials (PRT) are properly protected and device-bound
5. Whether BitLocker recovery keys are escrowed to Azure AD

## Bundled Validators

### 1. Device Join Status
Validates Azure AD / hybrid join configuration:
- Azure AD Joined status (dsregcmd + registry fallback)
- On-premises Domain Joined status (informational)
- Device Join Type classification (AAD/hybrid/unjoined)
- Tenant Information presence (TenantId, TenantName)
- Device Authentication status

### 2. Windows Hello for Business
Validates passwordless authentication readiness:
- WHfB Policy enabled (GPO/MDM)
- NGC Credential Provider registered
- PIN Complexity requirements (minimum 6 digits)
- NGC Key Container with enrolled keys
- Biometric hardware availability (informational)

### 3. Intune/MDM Enrollment
Validates mobile device management enrollment:
- MDM Enrollment detection via registry
- MDM Authority identification (Intune / third-party)
- Compliance Policies applied via PolicyManager
- Configuration Profiles deployed via IME

### 4. Cloud Credential Protection
Validates Primary Refresh Token and SSO security:
- PRT Status (AzureAdPrt : YES)
- PRT Update Time (active token refresh)
- Cloud Kerberos Trust for passwordless sign-in
- Device-Bound PRT via NGC key
- SSO State completeness (PRT + Authority)

### 5. BitLocker Cloud Escrow
Validates disk encryption with cloud recovery:
- BitLocker protection enabled on C: drive
- Recovery Key backed up to Azure AD
- XTS-AES encryption method (128 or 256-bit)
