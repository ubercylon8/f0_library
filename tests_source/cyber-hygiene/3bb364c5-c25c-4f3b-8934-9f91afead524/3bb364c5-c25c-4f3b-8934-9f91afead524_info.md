## Description

This cyber-hygiene test validates that Windows account lockout and password policies are properly configured to prevent brute force attacks. Microsoft blocks 7,000 password attacks per second - proper account lockout policies are essential for defense against credential-based attacks.

The test performs **READ-ONLY** validation of 5 critical policy settings against CIS Level 1 benchmarks, providing immediate visibility into the system's defensive posture against brute force, password guessing, and password spraying attacks.

## Technical Details

### Attack Flow

This test does NOT simulate an attack - it validates defensive configurations.

1. **Phase 0: Pre-flight Checks**
   - Verify administrator privileges (required for `net accounts`)
   - Exit with code 999 if not running as admin

2. **Phase 1: Query Account Policies**
   - Execute `net accounts` command
   - Parse lockout threshold, duration, and reset counter
   - Parse minimum password length
   - Execute `secedit /export` for password complexity

3. **Phase 2: Policy Evaluation**
   - Compare each setting against CIS Level 1 thresholds
   - Generate compliance status for each check
   - Display formatted results table

4. **Phase 3: Final Evaluation**
   - Determine overall compliance (all 5 must pass)
   - Report COMPLIANT (126) or NON-COMPLIANT (101)
   - Provide specific remediation commands if non-compliant

### Policy Validation Details

| Policy | Query Method | Compliant Value | Non-Compliant Value |
|--------|--------------|-----------------|---------------------|
| Lockout Threshold | `net accounts` | 1-5 attempts | 0 (disabled) or >5 |
| Lockout Duration | `net accounts` | >= 15 minutes | < 15 minutes |
| Reset Counter After | `net accounts` | >= 15 minutes | < 15 minutes |
| Min Password Length | `net accounts` | >= 14 chars | < 14 chars |
| Password Complexity | `secedit /export` | Enabled (1) | Disabled (0) |

## Expected Results

### Compliant System (Code 126)

All 5 policy checks pass CIS Level 1 thresholds:
- Lockout threshold is 1-5 attempts
- Lockout duration >= 15 minutes
- Reset counter >= 15 minutes
- Minimum password length >= 14 characters
- Password complexity is enabled

**Interpretation**: The system has basic brute force protections in place.

### Non-Compliant System (Code 101)

One or more policy checks fail:
- Lockout threshold is 0 (disabled) or > 5
- Lockout duration < 15 minutes
- Reset counter < 15 minutes
- Minimum password length < 14 characters
- Password complexity is disabled

**Interpretation**: The system is vulnerable to brute force attacks. Immediate remediation recommended.

### Test Error (Code 999)

Test cannot complete:
- Not running as Administrator
- `net accounts` command failed
- `secedit` export failed

## Remediation Commands

If the test reports NON-COMPLIANT, run these commands as Administrator:

```cmd
:: Set lockout threshold to 5 attempts
net accounts /lockoutthreshold:5

:: Set lockout duration to 15 minutes
net accounts /lockoutduration:15

:: Set reset counter to 15 minutes
net accounts /lockoutwindow:15

:: Set minimum password length to 14 characters
net accounts /minpwlen:14
```

For password complexity, use Group Policy Editor (`gpedit.msc`):
1. Navigate to: Computer Configuration > Windows Settings > Security Settings > Account Policies > Password Policy
2. Enable "Password must meet complexity requirements"
3. Run `gpupdate /force` to apply changes
