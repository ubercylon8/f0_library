# Local Administrator Password Solution (LAPS) Validator

## Test Information

**Test ID**: 05c39526-5374-419f-bf1e-68468400f3c6
**Test Name**: Local Administrator Password Solution (LAPS) Validator
**Category**: Cyber Hygiene / Configuration Validation
**Severity**: High
**MITRE ATT&CK**: T1078.003, T1021.002, T1550.002

## Description

This cyber hygiene test validates that Local Administrator Password Solution (LAPS) is properly configured on Windows endpoints. LAPS automatically manages and rotates local administrator passwords, storing them securely in Active Directory or Azure AD.

**Why LAPS Matters:**

The NSA/CISA Top 10 Cybersecurity Misconfigurations identifies "poor credential hygiene" as a critical security gap. When organizations deploy systems with shared local administrator passwords (often the same password set during imaging), attackers can:

1. **Compromise one system** using the local admin password
2. **Move laterally** to every other system with the same password
3. **Establish persistence** across the entire network
4. **Escalate privileges** by accessing privileged workstations

LAPS eliminates this attack vector by:
- Automatically generating unique, random passwords for each system
- Rotating passwords on a configurable schedule
- Storing passwords securely in AD/Azure AD with role-based access
- Providing audit logging for password retrievals

This test validates two LAPS implementations:
- **Windows LAPS**: Built-in to Windows 11/Server 2019+ (April 2023 update)
- **Legacy LAPS**: Microsoft's original LAPS requiring AD schema extension

## Technical Details

### Validation Flow

1. **Phase 0: Initialization**
   - Verify administrator privileges (required for registry access)
   - Check if system is a Domain Controller (LAPS doesn't apply)
   - Initialize Schema v2.0 compliant logging

2. **Phase 1: Windows LAPS Cmdlet Check**
   - Execute PowerShell: `Get-Command -Name Get-LapsDiagnostics`
   - If cmdlet exists, Windows LAPS feature is available
   - Requires Windows 11 22H2+ or Server 2019+ with April 2023 update

3. **Phase 2: Windows LAPS GPO Check**
   - Open registry key: `HKLM\SOFTWARE\Policies\Microsoft\Windows\LAPS`
   - Read `BackupDirectory` value (0=disabled, 1=Azure AD, 2=AD)
   - If key exists and BackupDirectory != 0, Windows LAPS is configured
   - Also reads password policy settings (informational)

4. **Phase 3: Legacy LAPS Enabled Check**
   - Open registry key: `HKLM\SOFTWARE\Policies\Microsoft Services\AdmPwd`
   - Read `AdmPwdEnabled` value (1=enabled)
   - Also reads password policy settings (informational)

5. **Phase 4: Legacy LAPS CSE Check**
   - Check for file: `C:\Program Files\LAPS\CSE\AdmPwd.dll`
   - Check for file: `C:\Program Files (x86)\LAPS\CSE\AdmPwd.dll`
   - Check for CSE GUID in registry: `{D76B9641-3288-4f75-942D-087DE603E3EA}`

6. **Phase 5: Password Policy Check (Informational)**
   - Evaluate password length against 14+ character recommendation
   - Evaluate password age against 30 day maximum recommendation
   - Log warnings for suboptimal settings

7. **Phase 6: Compliance Determination**
   - Windows LAPS configured = (Cmdlet Available AND GPO Configured)
   - Legacy LAPS configured = (AdmPwdEnabled=1 AND CSE Installed)
   - COMPLIANT = (Windows LAPS configured OR Legacy LAPS configured)

### Key Indicators

Security teams monitoring for configuration validation activity may observe:

- Registry reads to `HKLM\SOFTWARE\Policies\Microsoft\Windows\LAPS`
- Registry reads to `HKLM\SOFTWARE\Policies\Microsoft Services\AdmPwd`
- PowerShell execution with `Get-Command` cmdlet
- File system access to `C:\Program Files\LAPS\CSE\`

## Detection Opportunities

1. **Registry Access Monitoring**
   - Monitor for read access to LAPS policy registry keys
   - Could indicate reconnaissance activity

2. **PowerShell Logging**
   - Script block logging captures the cmdlet availability check
   - Command line logging shows Get-Command execution

3. **File System Monitoring**
   - Access to LAPS CSE directory
   - Unusual for standard applications

## Expected Results

### Compliant System (Code 126)

At least one LAPS solution is properly configured:

**Windows LAPS Compliant**:
- Get-LapsDiagnostics cmdlet available
- BackupDirectory registry value != 0
- Passwords backed up to Azure AD or Active Directory

**Legacy LAPS Compliant**:
- AdmPwdEnabled = 1 in registry
- CSE DLL installed or GUID registered
- Passwords managed via Group Policy

### Non-Compliant System (Code 101)

Neither LAPS solution is configured:
- No Windows LAPS GPO applied
- No Legacy LAPS installation
- Local admin passwords may be shared across systems
- Lateral movement attacks possible

### Test Error (Code 999)

The test could not complete due to:
- Insufficient privileges (must run as Administrator)
- Timeout exceeded (rare for this test)
- Unexpected system error

## LAPS Types Comparison

| Feature | Windows LAPS | Legacy LAPS |
|---------|--------------|-------------|
| Availability | Windows 11 22H2+, Server 2019+ | All Windows versions |
| AD Schema Extension | Not required | Required |
| Installation | Built-in (April 2023 update) | Separate download |
| Azure AD Support | Yes | No |
| Managed Accounts | Built-in Admin + custom accounts | Built-in Admin only |
| Password History | Supported | Not supported |
| Encryption | AES-256 encrypted in AD | Clear text in AD attribute |
