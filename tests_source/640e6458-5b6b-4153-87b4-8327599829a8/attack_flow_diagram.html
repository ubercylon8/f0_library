Stage 1: EDR Process Discovery

MITRE ATT&CK: T1016

Technical Implementation

  • Creates process snapshot using CreateToolhelp32Snapshot
  • Enumerates all running processes via Process32FirstW/NextW
  • Compares process names against 40+ known EDR products
  • Extracts process IDs for subsequent targeting
CreateToolhelp32Snapshot(TH32CS_SNAPPROCESS, 0) Process32FirstW(snapshot, &processEntry) // Check against EDR list: msmpeng.exe, csagent.exe, etc.

Targeted EDR Products

  • Microsoft Defender & Defender for Endpoint
  • CrowdStrike Falcon
  • SentinelOne
  • Carbon Black Response/Defense
  • Cylance, Symantec, McAfee/Trellix
  • Trend Micro, Palo Alto Cortex XDR
  • FireEye/Mandiant, Sophos, ESET
  • Kaspersky, Tanium, Cisco Secure Endpoint

Detection Opportunities

• Process enumeration patterns targeting security products
• Unusual access to process information
• Sysmon Event ID 10 (Process Access)
• Behavioral detection of EDR targeting

Stage 2: WFP Network Isolation

MITRE ATT&CK: T1562.004

WFP Implementation Details

  • Re-enumerates EDR processes to get full executable paths
  • Initializes Windows Filtering Platform engine
  • Creates custom provider and sublayer for filter management
  • Converts process paths to AppID blobs using FwpmGetAppIdFromFileName0
  • Creates 4 blocking filters per EDR process:
  • IPv4 Outbound (FWPM_LAYER_ALE_AUTH_CONNECT_V4)
  • IPv6 Outbound (FWPM_LAYER_ALE_AUTH_CONNECT_V6)
  • IPv4 Inbound (FWPM_LAYER_ALE_AUTH_RECV_ACCEPT_V4)
  • IPv6 Inbound (FWPM_LAYER_ALE_AUTH_RECV_ACCEPT_V6)
FwpmEngineOpen0(NULL, RPC_C_AUTHN_DEFAULT, NULL, NULL, &engineHandle) FwpmGetAppIdFromFileName0(processPath, &appId) FwpmFilterAdd0(engineHandle, &filter, NULL, &filterId) // Filter action: FWP_ACTION_BLOCK | FWP_ACTION_FLAG_TERMINATING

Impact on EDR

  • EDR processes continue running (appears normal)
  • All cloud connectivity blocked silently
  • Telemetry uploads fail without alerts
  • Threat intelligence updates blocked
  • Remote management capabilities disabled

Detection Opportunities

• Windows Event IDs: 5441, 5157, 5152
• WFP provider creation with unusual names
• Filter application targeting security products
• Sudden loss of EDR cloud connectivity
• Administrative WFP API usage

Stage 3: Service Disruption

MITRE ATT&CK: T1489, T1562.001

Service Control Operations

  • Connects to Windows Service Control Manager
  • Opens handles to EDR/AV services
  • Attempts to stop each identified service
  • Tries to disable service auto-start configuration
  • Logs protection responses (Access Denied indicates protection)
mgr.Connect() s := mgr.OpenService(serviceName) s.Control(svc.Stop) s.UpdateConfig(mgr.Config{StartType: mgr.StartDisabled})

Targeted Services

  • Windows Defender: WinDefend, Sense, WdNisSvc
  • CrowdStrike: CSAgent, CSFalconService
  • SentinelOne: SentinelAgent, SentinelStaticEngine
  • Carbon Black: CarbonBlack, RepMgr
  • And many more mapped EDR services...

Detection Opportunities

• Service Control Manager API abuse
• Windows Event IDs: 7035, 7036, 4656, 4658
• Multiple service stop attempts in sequence
• Access denied errors (indicates protection)
• Service configuration change attempts
