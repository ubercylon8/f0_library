## Purpose

This test validates LimaCharlie's `--timeout` parameter functionality:

1. **Problem**: Long-running F0RT1KA tests exceed LC's default timeout (~3-5 min)
2. **Symptom**: RECEIPT events show exit code 259 (STILL_ACTIVE) instead of actual results
3. **Solution**: Use `--timeout <seconds>` parameter on the `run` command
4. **Validation**: This test confirms whether extended timeouts work correctly
