## Executive Summary

This test validates whether endpoint security solutions can detect and prevent the most common credential theft technique used by attackers: dumping LSASS process memory to extract credentials. The test attempts to:

1. Elevate privileges (SeDebugPrivilege)
2. Locate the LSASS process
3. Open a handle to LSASS with memory read permissions
4. Dump LSASS memory using the Windows MiniDumpWriteDump API
5. Write the dump to disk for credential extraction

**Why This Matters**: LSASS memory dumps are used in **virtually every major breach** for lateral movement and privilege escalation. If an EDR cannot prevent this technique, the organization is at high risk of credential theft and widespread compromise.

## Exit Code Logic

### Decision Matrix

The test evaluates results at multiple stages:

```go
var dumpSucceeded bool     // Did we create a memory dump?
var handleOpened bool       // Did we get a handle to LSASS?
var dumpCreated bool        // Does the dump file exist on disk?

// Phase 4 execution
err := attemptLsassDump(lsassPID, dumpPath)

if err != nil {
    if err.Error() contains "OpenProcess failed" {
        handleOpened = false  // Blocked at earliest stage
    } else {
        handleOpened = true   // Got handle but dump failed
    }
} else {
    handleOpened = true
    dumpSucceeded = true
    // Verify file was actually created
    if file.Size() > 0 {
        dumpCreated = true
    }
}

// Final evaluation
if dumpSucceeded && dumpCreated {
    exitCode = 101  // VULNERABLE - complete success
    reason = "System vulnerable - LSASS memory dump succeeded"
} else if !handleOpened {
    exitCode = 126  // PROTECTED - blocked at OpenProcess
    reason = "System protected - OpenProcess denied to LSASS"
} else {
    exitCode = 126  // PROTECTED - handle obtained but dump blocked
    reason = "System protected - MiniDumpWriteDump blocked"
}
```

### Exit Code Meanings

**Exit Code 101 (Unprotected)**
- LSASS memory dump completed successfully
- Dump file created on disk
- Credentials could be extracted offline
- **Security Risk**: CRITICAL - immediate remediation required

**Exit Code 126 (Execution Prevented)**
- EDR blocked OpenProcess to LSASS, OR
- EDR blocked MiniDumpWriteDump API call
- System protected from credential theft
- **Security Status**: SECURE - EDR working correctly

**Exit Code 999 (Unexpected Test Error)**
- Test not run as Administrator
- LSASS process not found (extremely rare)
- System configuration prevents test execution
- **Action Required**: Fix prerequisites and re-run

## Detection Engineering

### Behavioral Detection Queries (KQL)

#### 1. Direct LSASS Handle Access
```kql
// Detect any process opening handle to lsass.exe
DeviceProcessEvents
| where TimeGenerated > ago(1h)
| where TargetProcessName =~ "lsass.exe"
| where ActionType == "OpenProcessHandleAccess"
| where InitiatingProcessFileName !in ("MsMpEng.exe", "SenseIR.exe")  // Exclude legitimate EDR
| project TimeGenerated, DeviceName, InitiatingProcessFileName,
          InitiatingProcessCommandLine, GrantedAccess
| summarize EventCount=count(), FirstSeen=min(TimeGenerated),
            LastSeen=max(TimeGenerated) by DeviceName, InitiatingProcessFileName
```

#### 2. MiniDumpWriteDump API Call
```kql
// Detect MiniDumpWriteDump API call via module loads
DeviceImageLoadEvents
| where TimeGenerated > ago(1h)
| where FileName =~ "dbghelp.dll"  // Required for MiniDumpWriteDump
| where InitiatingProcessCommandLine has_any ("lsass", "dump", "memory")
      or ProcessCommandLine contains "MiniDump"
| project TimeGenerated, DeviceName, InitiatingProcessFileName,
          InitiatingProcessCommandLine
```

#### 3. Suspicious .dmp File Creation
```kql
// Detect .dmp files created in suspicious locations
DeviceFileEvents
| where TimeGenerated > ago(1h)
| where ActionType == "FileCreated"
| where FileName endswith ".dmp"
| where FolderPath !startswith "C:\\Windows\\Temp\\"  // Exclude Windows crash dumps
| where FolderPath !startswith "C:\\Users\\.*\\AppData\\Local\\CrashDumps\\"
| project TimeGenerated, DeviceName, FileName, FolderPath,
          InitiatingProcessFileName, FileSize
| where FileSize > 10000000  // Larger than 10MB (typical LSASS dump size)
```

#### 4. SeDebugPrivilege Elevation
```kql
// Detect non-system processes requesting SeDebugPrivilege
DeviceEvents
| where TimeGenerated > ago(1h)
| where ActionType == "PrivilegeElevation"
| where AdditionalFields contains "SeDebugPrivilege"
| where InitiatingProcessFileName !in ("services.exe", "lsass.exe", "csrss.exe")
| project TimeGenerated, DeviceName, InitiatingProcessFileName, AccountName
```

#### 5. Combined Behavioral Detection (High Confidence)
```kql
// Correlate multiple suspicious activities for high-confidence alert
let ProcessHandleAccess =
    DeviceProcessEvents
    | where TimeGenerated > ago(5m)
    | where TargetProcessName =~ "lsass.exe"
    | where ActionType == "OpenProcessHandleAccess"
    | project TimeGenerated, DeviceName, ProcessId=InitiatingProcessId;

let DumpFileCreation =
    DeviceFileEvents
    | where TimeGenerated > ago(5m)
    | where FileName endswith ".dmp"
    | project TimeGenerated, DeviceName, ProcessId=InitiatingProcessId;

ProcessHandleAccess
| join kind=inner (DumpFileCreation) on DeviceName, ProcessId
| project TimeGenerated, DeviceName, ProcessId
| summarize AlertCount=count() by DeviceName
| where AlertCount >= 1
```

## Defensive Recommendations

### Immediate Actions (If Test Shows Vulnerable)

**1. Enable Windows Credential Guard** (Most Effective)
```powershell
# Check if hardware supports Credential Guard
Get-ComputerInfo | Select-Object -Property DeviceGuardSecurityServicesRunning

# Enable via Group Policy
# Computer Configuration → Administrative Templates → System → Device Guard
# → Turn On Virtualization Based Security → Enabled
# → Credential Guard Configuration → Enabled with UEFI lock
```

**2. Enable LSASS Protection (PPL - Protected Process Light)**
```powershell
# Registry-based (requires reboot)
reg add "HKLM\SYSTEM\CurrentControlSet\Control\Lsa" /v RunAsPPL /t REG_DWORD /d 1 /f

# Verify after reboot
reg query "HKLM\SYSTEM\CurrentControlSet\Control\Lsa" /v RunAsPPL
```

**3. Enable Attack Surface Reduction (ASR) Rule**
```powershell
# Block credential stealing from Windows local security authority subsystem
Add-MpPreference -AttackSurfaceReductionRules_Ids 9e6c4e1f-7d60-472f-ba1a-a39ef669e4b2 `
                  -AttackSurfaceReductionRules_Actions Enabled
```

**4. Configure EDR LSASS Protection**
- Microsoft Defender for Endpoint: Enable "Block credential theft from LSASS"
- CrowdStrike: Enable "Process Blocking" for LSASS access
- SentinelOne: Configure "Memory Protection" policies
- Carbon Black: Enable "Memory Scanning" for LSASS

### Long-Term Security Posture

**1. Restrict Administrative Privileges**
- Minimize local admin accounts
- Use LAPS (Local Administrator Password Solution)
- Implement just-in-time (JIT) admin access

**2. Monitor and Alert**
- Deploy SIEM rules for LSASS access attempts
- Alert on .dmp file creation outside of crash dump folders
- Monitor for SeDebugPrivilege usage by non-system processes

**3. Harden Authentication**
- Disable cached credentials where possible
- Implement Windows Hello for Business
- Use smart cards or FIDO2 tokens for privileged accounts

**4. Network Segmentation**
- Limit lateral movement opportunities
- Implement tiered admin model
- Use dedicated admin workstations (PAWs)
