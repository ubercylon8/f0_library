## Description

This test simulates the classic process injection technique using CreateRemoteThread, one of the most commonly observed methods used by malware to inject malicious code into legitimate processes. The test evaluates EDR/AV capabilities to detect and prevent each stage of the injection process, from initial process access to remote thread creation.

The injection technique is widely used by various malware families including banking trojans, RATs, and ransomware to hide malicious activity within legitimate processes, bypass application whitelisting, and escalate privileges.

## Technical Details

### Attack Flow
1. **Phase 1: Initialization**
   - Initialize test framework and logging
   - Verify administrative privileges
   - Create working directory (C:\F0)

2. **Phase 2: Target Process Setup**
   - Spawn notepad.exe as injection target
   - Obtain process ID for targeting
   - Wait for process initialization

3. **Phase 3: Process Injection Attempt**
   - **Step 1 - OpenProcess**: Acquire handle with PROCESS_CREATE_THREAD | PROCESS_VM_OPERATION | PROCESS_VM_WRITE permissions
   - **Step 2 - VirtualAllocEx**: Allocate executable memory in remote process
   - **Step 3 - WriteProcessMemory**: Write benign shellcode to allocated memory
   - **Step 4 - CreateRemoteThread**: Create thread to execute injected code

4. **Phase 4: Protection Evaluation**
   - Analyze which injection stage was blocked (if any)
   - Determine system protection status
   - Clean up resources and spawned process

### Key Indicators
- Process opening with VM_WRITE permissions
- Memory allocation with EXECUTE_READWRITE protection
- Cross-process memory write operations
- Remote thread creation in legitimate process
- Suspicious process handle access patterns

## Detection Opportunities

1. **Process Access Monitoring**
   - Detection of OpenProcess with injection-related permissions
   - Alert on PROCESS_VM_WRITE | PROCESS_CREATE_THREAD combination
   - Monitor for suspicious source-target process relationships

2. **Memory Operation Detection**
   - VirtualAllocEx calls with PAGE_EXECUTE_READWRITE
   - Cross-process memory allocation patterns
   - Detection of executable memory regions in remote processes

3. **Code Injection Indicators**
   - WriteProcessMemory operations to remote processes
   - Detection of shellcode patterns in memory
   - Monitoring for cross-process memory modifications

4. **Thread Creation Monitoring**
   - CreateRemoteThread API calls
   - New thread creation in existing processes
   - Thread start addresses outside normal module boundaries

## Implementation Notes

### Windows API Usage
The test uses native Windows APIs exactly as malware would:
```c
OpenProcess(PROCESS_ALL_ACCESS, FALSE, pid)
VirtualAllocEx(hProcess, NULL, size, MEM_COMMIT|MEM_RESERVE, PAGE_EXECUTE_READWRITE)
WriteProcessMemory(hProcess, addr, shellcode, size, &written)
CreateRemoteThread(hProcess, NULL, 0, addr, NULL, 0, &threadId)
```

### Payload Safety
- Uses benign message box shellcode
- No file system modifications
- No registry changes
- No network communication
- Automatic cleanup on completion

### Single-Binary Deployment
- Self-contained executable with all dependencies
- Automatic F0RT1KA certificate installation
- No external files required
- Clean deployment and execution
