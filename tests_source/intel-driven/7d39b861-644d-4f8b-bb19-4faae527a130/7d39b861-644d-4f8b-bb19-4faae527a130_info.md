## Description

This multi-stage test simulates the complete Agrius destructive attack chain targeting banking infrastructure. Agrius, also tracked as Pink Sandstorm, Agonizing Serpens, and BlackShadow, is an Iranian state-sponsored threat group linked to MOIS that has conducted confirmed wiper campaigns against Israeli financial sector organizations.

The test models the full killchain from initial webshell deployment through simultaneous multi-wiper deployment and anti-forensics, testing EDR detection capabilities at each stage. Three wiper variants are deployed concurrently: MultiLayer (random data overwrite), PartialWasher (header corruption), and BFG Agonizer (multi-pass aggressive wipe).

All wiper operations are SAFE simulations that only target test files created by the test itself in `c:\F0\wiper_test\`.

## Technical Details

### Attack Flow

1. **Stage 1: ASPXSpy Webshell Deployment** (T1505.003)
   - Deploys three simulated ASPXSpy webshell variants to c:\F0
   - File names: aspxspy.aspx, error5.aspx, contact.aspx
   - Monitors for EDR quarantine of .aspx files
   - Detection: Webshell content patterns, .aspx file creation in staging directories

2. **Stage 2: IPsec Helper Service Persistence** (T1543.003)
   - Creates a fake service binary at c:\F0\ipsec_helper_svc.exe
   - Attempts sc.exe create with auto-start and non-standard binary path
   - Queries service to confirm creation
   - Cleanup: Deletes service and binary after testing
   - Detection: sc.exe service creation with unknown binary path

3. **Stage 3: EDR Tampering** (T1562.001)
   - Deploys simulated GMER64.sys driver file
   - Attempts to create kernel-type service via sc.exe
   - Attempts to disable EDR services (Sense, WinDefend, CrowdStrike)
   - Attempts to stop running EDR services
   - Safety: Re-enables/restarts services immediately if successfully tampered
   - Detection: Driver loading, EDR service modification, GMER64.sys deployment

4. **Stage 4: Multi-Wiper Deployment** (T1485)
   - Creates 20 test files across 3 subdirectories (simulating banking data)
   - Launches three concurrent wiper simulations:
     - MultiLayer: Random data overwrite of transaction files
     - PartialWasher: 512-byte header corruption of customer data files
     - BFG Agonizer: 3-pass overwrite (0x00, 0xFF, 0xAA) + deletion of audit log files
   - Writes boot sector simulation marker (LOG ONLY, no actual MBR/VBR modification)
   - Detection: Mass file overwrite patterns, simultaneous process execution

5. **Stage 5: Anti-Forensics** (T1070.001)
   - Attempts to clear 5 Windows Event Log channels via wevtutil.exe
   - Creates and executes remover.bat (Agrius self-deletion pattern with ping delay)
   - Remover.bat cleans up test artifacts from previous stages
   - Detection: wevtutil.exe event log clearing, batch script execution

### Key Indicators
- ASPXSpy .aspx file creation (aspxspy.aspx, error5.aspx, contact.aspx)
- sc.exe service creation: F0RT1KA_IPsecHelper
- GMER64.sys file deployment to c:\F0

- Concurrent execution of multiple wiper processes

## Detection Opportunities

1. **Webshell Detection** (Stage 1)
   - .aspx file creation with webshell content patterns
   - Server-side scripting file in non-IIS staging directory
   - ASPXSpy-specific code patterns in file content

2. **Service Creation** (Stage 2)
   - sc.exe create with unknown binary path
   - Auto-start service pointing to c:\F0 directory
   - Service name pattern: IPsec Helper with unusual binary

3. **Driver Loading** (Stage 3)
   - GMER64.sys file deployment (known anti-rootkit tool abused by attackers)
   - Kernel-type service creation via sc.exe
   - EDR service modification (config, stop) via sc.exe

4. **Mass File Operations** (Stage 4)
   - High-volume file overwrite operations in short timeframe
   - Concurrent file modification across multiple directories
   - Multi-pass overwrite pattern (null, ones, alternating)
   - File deletion after overwrite

5. **Event Log Clearing** (Stage 5)
   - wevtutil.exe cl invocations
   - Security/System/Application log clearing
   - Batch script execution with ping-based delays
   - Self-deleting batch file pattern
