### Key Disclosures Modeled

From the GTIG report, HONESTCUE is described as:

> "In September 2025, GTIG observed malware samples tracked as HONESTCUE, leveraging Gemini's API to outsource functionality generation. HONESTCUE is a downloader and launcher framework that sends a prompt via Google Gemini's API and receives C# source code as the response. HONESTCUE shares capabilities similar to PROMPTFLUX's 'just-in-time' technique; however, rather than leveraging an LLM to update itself, HONESTCUE calls the Gemini API to generate code that operates the 'stage two' functionality."

The v2 test models the three behaviors GTIG identifies as core to HONESTCUE's operation:

1. **Gemini API prompt/response at runtime** — modeled by stage 1 via real HTTPS GET to `raw.githubusercontent.com` for a Gemini-shaped JSON response. v1 used a loopback mock server; v2 uses GitHub-raw as the trusted-hosting surface so the TLS+DNS+SNI IOCs are genuine.
2. **C# source code as LLM response** — modeled via pre-staged `gemini_response.json` with the exact Gemini API schema (`candidates[].content.parts[].text`) hosting the stage-2 C# source. All three GTIG HONESTCUE prompts are embedded as string constants in stage-1 source.
3. **Stage-two execution via compiled-and-loaded C# assembly** — modeled by stage 2 via `Microsoft.CodeAnalysis.CSharp` (Roslyn) `CSharpCompilation.Create(...).Emit(MemoryStream)` + `Assembly.Load(byte[])`. Real HONESTCUE uses .NET Framework 4.x `CSharpCodeProvider`; v2 uses Roslyn on .NET 8 as the modern-runtime equivalent.

## Why GitHub Raw Hosting?

v1 modeled HONESTCUE's Discord-CDN abuse with a hosts-file redirect + self-signed loopback server. v2 switches to `raw.githubusercontent.com` for the following reasons:

1. **Real network IOCs** — GitHub-raw requests produce authentic TLS handshakes, JA3/JA4 fingerprints, DNS EventID 22 entries, and SNI observations that EDR/NDR products can actually see. Loopback mock servers never reached the network stack.
2. **No hosts-file mutation** — simpler, safer, no admin context required, no system config change.
3. **No Discord-CDN risk** — some corporate egress filters block or alert on `cdn.discordapp.com`; GitHub raw is usually allowed for developer productivity reasons.
4. **Mirrors real TA tradecraft** — recent threat actor campaigns (e.g., LAZARUS, FIN7) increasingly use GitHub raw URLs as payload-staging infrastructure for the same trust-signal reason HONESTCUE uses trusted CDNs.
5. **Portable across lab environments** — operators upload two files once to any GitHub-raw-capable repo; the test auto-discovers them via the constants.
