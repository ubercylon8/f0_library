## Description

This test simulates the HONESTCUE malware family disclosed by the Google Threat Intelligence Group (GTIG) in its February 2026 report "GTIG AI Threat Tracker: Distillation, Experimentation, and (Continued) Integration of AI for Adversarial Use." HONESTCUE is notable because it treats a commercial LLM API (Google's Gemini) as a *runtime dependency* â€” not as a developer's coding assistant but as a live provider of stage-two C# source code, compiled and executed on the victim machine each time the malware runs. GTIG compared HONESTCUE's "just-in-time code" technique to the earlier PROMPTFLUX family, noting that HONESTCUE uses the Gemini API to generate code operating the "stage two" functionality rather than updating itself.

The test models the full chain in three stages:

1. **Stage 1 (T1071.001)** issues a HONESTCUE-shaped HTTPS POST to a loopback mock Gemini endpoint pinned by self-signed cert, receives C# source in a Gemini-schema response, and hands off the source on disk.
2. **Stage 2 (T1027.004 + T1027.010 + T1620)** uses `powershell.exe` + `CSharpCodeProvider` (`GenerateInMemory=true`) to compile the LLM-sourced C#, reflectively loads the result via `Assembly.Load(byte[])`, and invokes the entry point which reads a Defender registry subkey and writes a marker in `ARTIFACT_DIR`.
3. **Stage 3 (T1105 + T1583.006 + T1565.001)** modifies the hosts file to redirect `cdn.discordapp.com` to a loopback Discord-CDN lookalike, fetches an UNSIGNED PE and an executable payload, drops them to `%TEMP%`, and executes. Hosts-file restoration is guaranteed on every exit path.

## Subcategory Note

This test uses `SUBCATEGORY: apt` for compatibility with the current CLAUDE.md enum. We **recommend extending the official enum to include `llm-abuse`** for AI-as-runtime-component threats where a cloud LLM API is a live dependency at execution time rather than a development assistant. HONESTCUE, PROMPTFLUX, and their successors represent a distinct TTP category with distinct detection surfaces (LLM API egress patterns, non-browser client TLS signatures, runtime code compilation triggered by LLM responses) that deserve first-class categorization. Operators should evaluate whether to add `llm-abuse` to the official subcategory enum.
