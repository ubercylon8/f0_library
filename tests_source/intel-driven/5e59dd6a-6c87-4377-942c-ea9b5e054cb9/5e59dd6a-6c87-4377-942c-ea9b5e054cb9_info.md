## Purpose

Give EDR/AV products a safe, controlled workload that exercises the observable API surface of BlueHammer's full kill-chain primitives — including the SAM-hive read API surface that BlueHammer's PoC ultimately uses, but executed against a **synthetic sandbox hive** rather than any real Windows hive. The goal is to let defenders measure detection coverage on every primitive BlueHammer touches, with strict safety guarantees ensuring no real credential material is ever accessed.

## MITRE mapping rationale

**T1211 — Exploitation for Defense Evasion** (stages 1, 3 Part A)
- Cloud Files sync-root registration with a fetch-placeholder callback (PoC uses this to intercept Defender's scan path)
- NT Object Manager enumeration of `\Device` for VSS shadow copies (PoC uses this to find a snapshot to attack)

**T1562.001 — Impair Defenses: Disable or Modify Tools** (stage 2 + stage 3 Part B)
- `FSCTL_REQUEST_BATCH_OPLOCK` on a file (PoC uses against a Defender scan path; we use against a sandbox file)
- Read-only WMI Win32_ShadowCopy enumeration (PoC reconnoiters before VSS abuse; we exercise the WQL signal without proceeding)

**T1003.002 — OS Credential Dumping: Security Account Manager** (stage 4)
- `RegLoadKey` + `RegOpenKeyEx(KEY_READ)` — the API signature BlueHammer uses to read SAM. We execute this against a **synthetic sandbox hive we generated at runtime**, never against a real SAM hive. The detection telemetry an EDR sees is identical; the credential exposure is zero.

**T1134.001 — Access Token Manipulation: Token Impersonation/Theft** (stage 4)
- `AdjustTokenPrivileges` enabling `SeBackupPrivilege` and `SeRestorePrivilege` — the privilege-enable telemetry signal that precedes any SAM-hive load attempt.

## Detailed stage behavior

### Stage 1 — `T1211-cfapi` (Cloud Files sync-root + Mimikatz-named EICAR drop)

1. Creates `C:\Users\fortika-test\BlueHammerSandbox\cfapi-root\`
2. `CfRegisterSyncRoot(sandboxPath, {ProviderName="F0RT1KA-BLUEHAMMER-SIM", ProviderVersion="1.0"}, policies, CF_REGISTER_FLAG_NONE)`
3. `CfConnectSyncRoot(sandboxPath, callbacks[CF_CALLBACK_TYPE_FETCH_PLACEHOLDERS -> no-op Go callback], ...)`
4. **(B2)** Drops EICAR test string to `Mimikatz_dump_Win32_signature.txt` — content is EICAR (AMSI signal); filename suggests Mimikatz (path-rule signal). Both detection paths fire.
5. Sleep 3s, `cmd /c dir <sandbox>` to surface a listing event
6. `CfDisconnectSyncRoot` + `CfUnregisterSyncRoot` cleanup unconditionally

### Stage 2 — `T1562.001-oplock` (Batch oplock on sandbox file)

Unchanged from v1. `FSCTL_REQUEST_BATCH_OPLOCK` on a sandbox file with hard 5-second timeout; oplock always released regardless of break/timeout.

### Stage 3 — `T1211-vssenum` (NT enum + WMI enum + transacted-open)

Part A unchanged (NT Object Manager enumeration of `\Device`).

**(B3) Part B is new:** WMI Win32_ShadowCopy enumeration via PowerShell `Get-WmiObject -Class Win32_ShadowCopy`. The query is purely enumerative — `Win32_ShadowCopy.Create()` and `.Delete()` are NEVER called. PoC fidelity deviation: the real PoC uses direct `IWbemServices->ExecQuery` COM; this test invokes the same provider via PowerShell for implementation simplicity. The sensor-visible event is identical.

Part C (transacted-open probe on sandbox file) unchanged.

### Stage 4 — `T1003.002-samsim` (Privilege-enable + synthetic SAM-hive load) **NEW in v2**

1. **(B4)** `OpenProcessToken(GetCurrentProcess(), TOKEN_ADJUST_PRIVILEGES|TOKEN_QUERY)`
2. **(B4)** `LookupPrivilegeValueW("SeBackupPrivilege")` + `AdjustTokenPrivileges(SeBackupPrivilege, ENABLED)` — telemetry regardless of OS grant/deny
3. **(B4)** Same for `SeRestorePrivilege` — required to proceed to part B
4. **Standard-user / non-admin path:** if `SeRestorePrivilege` was not granted, the OS returns `ERROR_NOT_ALL_ASSIGNED`. The privilege-denial telemetry signal is logged and the stage exits success (B4 was the value). **No hive load is attempted.**
5. **Admin / elevated path:** stage proceeds to part B:
   - **(B5)** `RegCreateKeyEx(HKLM\SOFTWARE\F0_BLUEHAMMER_SAMSIM_TEMP)` — creates a sandbox temp key
   - **(B5)** `RegSaveKey(tempKey, ARTIFACT_DIR\sam-sim.hiv)` — writes a synthetic hive file
   - **(B5)** Watchdog goroutine starts (30s timeout). If the hive remains loaded past timeout, it is force-unloaded.
   - **(B5)** `RegLoadKey(HKLM, "F0_BLUEHAMMER_SAMSIM", sam-sim.hiv)` — loads the SYNTHETIC sandbox hive under a unique mount name
   - **(B5)** `RegOpenKeyEx(HKLM\F0_BLUEHAMMER_SAMSIM, KEY_READ)` — the SAM-hive-read API signature
   - `RegCloseKey` + `RegUnLoadKey` (defer LIFO order)
   - Watchdog stop + defer cleanup of temp HKLM key + sandbox file

**Safety guarantees for stage 4:**
- The hive loaded under HKLM\F0_BLUEHAMMER_SAMSIM is a **synthetic sandbox file** the test created, not any real Windows hive
- The mount name is unique to this test (contains "BLUEHAMMER")
- Watchdog (30s) force-unloads the hive if any subsequent step hangs
- Temp HKLM key is removed on cleanup (defer, LIFO)
- Sandbox hive file is removed on cleanup (defer)
- Standard-user execution skips part B entirely
