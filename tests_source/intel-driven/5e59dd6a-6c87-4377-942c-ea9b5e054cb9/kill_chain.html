This test measures only detection coverage on the early detectable primitives - the ones that fire before an EDR has committed to letting the chain continue.

✅ Protected (Exit 126)

EDR detected and blocked at least one early BlueHammer primitive - most likely the non-OneDrive CfRegisterSyncRoot call (stage 1), the batch-oplock on a non-backup-context process (stage 2), or the \Device enumeration + CreateFileTransactedW combo (stage 3). An attacker running the real PoC against this host would be stopped before reaching the Phase 4 TOCTOU race.

⚠ Unprotected (Exit 101)

All 3 early primitives executed without EDR intervention. A real attacker running an unmodified BlueHammer binary could continue into the credential-access phases (SAM leak → NTLM decrypt → SYSTEM shell) without tripping an early alert. This test does NOT execute those phases - but their detectability is gated on stopping stages 1-3.
