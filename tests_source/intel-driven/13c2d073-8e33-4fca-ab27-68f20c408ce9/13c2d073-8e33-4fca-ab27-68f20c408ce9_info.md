## Description

This test simulates the complete APT33 Tickler backdoor attack chain as documented in threat intelligence reports on Peach Sandstorm operations from 2024-2025. The Tickler backdoor represents APT33's evolution from simple spearphishing to sophisticated multi-stage attacks leveraging DLL sideloading, dual persistence mechanisms, binary masquerading, and Azure-hosted C2 infrastructure.

The test implements a 6-stage sequential killchain where each stage maps to a distinct MITRE ATT&CK technique. If any stage is blocked by EDR, the test halts and reports which specific technique was detected, providing technique-level detection precision for security operations teams.

## Technical Details

### Attack Flow

1. **Stage 1: Spearphishing Attachment (T1566.001)**
   - Creates ZIP archive named `Q3_Financial_Report_2025.pdf.zip` (double extension masquerading)
   - ZIP contains simulated Tickler backdoor (`Microsoft.SharePoint.NativeMessaging.exe`), sideloaded DLLs (`msvcp140.dll`, `vcruntime140.dll`), and decoy PDF
   - Extracts contents to `c:\Users\fortika-test\tickler_extract\` (non-whitelisted path, EDR-visible)

2. **Stage 2: DLL Side-Loading (T1574.002)**
   - Copies legitimate Microsoft signed binary (notepad.exe) renamed as `Microsoft.SharePoint.NativeMessaging.exe`
   - Places simulated sideloading DLLs in same directory
   - Executes the renamed binary, triggering EDR sideloading detection patterns
   - Tests whether EDR detects VC++ runtime DLLs loaded from non-standard paths

3. **Stage 3: Registry Run Keys (T1547.001)**
   - Detects SYSTEM vs user context using `whoami` (Rule 2 compliance)
   - SYSTEM context: writes to `HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run`
   - User context: writes to `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`
   - Value name: `SharePoint` pointing to sideloaded binary in non-standard path
   - Saves original state for cleanup restoration

4. **Stage 4: Scheduled Task (T1053.005)**
   - Creates scheduled task `MicrosoftSharePointSync` with ONLOGON trigger
   - Points to the masqueraded binary with HIGHEST run level
   - Provides redundant persistence alongside registry Run key
   - Saves task existence state for cleanup

5. **Stage 5: Masquerading (T1036)**
   - Creates `SharePoint.exe` copy of the backdoor binary
   - Executes from non-standard path (not Program Files)
   - Tests EDR detection of binary version info mismatch (original name vs file name)
   - Calculates and logs SHA256 hash for forensic tracking

6. **Stage 6: Web Protocols C2 (T1071.001)**
   - Collects system fingerprint (hostname, username, IP, computer name)
   - Base64-encodes fingerprint data (models Tickler encoding)
   - Stages encoded beacon data to `c:\F0\c2_staging\beacon.dat`
   - Attempts HTTP POST to non-standard ports 808 and 880 (Tickler C2 ports)
   - Uses custom User-Agent: `Microsoft SharePoint/16.0`
   - Targets localhost (safe -- no actual exfiltration)

### Key Indicators

- DLL loading from non-standard paths (msvcp140.dll, vcruntime140.dll)
- Registry Run key `SharePoint` referencing executable outside Program Files
- Scheduled task `MicrosoftSharePointSync` with ONLOGON trigger
- Binary named `Microsoft.SharePoint.NativeMessaging.exe` or `SharePoint.exe` running from user directory
- HTTP POST requests on ports 808 and 880 with base64-encoded payloads
- ZIP archives with double extensions (`.pdf.zip`)
- User-Agent string `Microsoft SharePoint/16.0` from non-browser process

## Detection Opportunities

1. **DLL Sideloading Patterns** -- VC++ runtime DLLs loaded from non-standard directories by renamed Microsoft binaries
2. **Registry Persistence Anomalies** -- Run key additions with SharePoint-related names pointing to non-standard paths
3. **Scheduled Task Creation** -- schtasks.exe creating tasks referencing SharePoint binaries with high privileges
4. **Binary Masquerading** -- Executables with Microsoft-sounding names running from user directories with version info mismatches
5. **C2 Communication** -- HTTP POST traffic on non-standard ports (808, 880) with base64-encoded payloads
6. **Spearphishing Delivery** -- ZIP file creation with double extensions in user download areas
7. **Combined Behavioral Correlation** -- Multiple indicators appearing on same host within short timeframe

## Cleanup

Embedded cleanup utility (`cleanup_utility.exe`) runs unattended and:
1. Restores registry Run key to original state
2. Removes scheduled task `MicrosoftSharePointSync`
3. Deletes spearphishing artifacts and extracted files
4. Removes C2 staging data
5. Cleans up masqueraded binaries
6. Removes stage binaries and log files
