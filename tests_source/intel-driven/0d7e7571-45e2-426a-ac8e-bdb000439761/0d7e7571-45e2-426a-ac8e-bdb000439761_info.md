### Stage Timeout Budgets (v2.1 §3d Operational Hygiene)

Per-stage execution-time budgets enforced by the orchestrator's `time.AfterFunc` watchdog (see `executeKillchainStage` in `0d7e7571-45e2-426a-ac8e-bdb000439761.go`). Budgets are sized at ~5–10× lab-observed runtime to leave headroom for slow scheduler stalls. The R5 budget is the most consequential — `CoCreateInstance` against the CFAPI Sync Root Manager broker can plausibly hang under contention, and the watchdog is the only recovery path.

| Stage | Technique | Budget | Lab-observed | Reason |
|---|---|---|---|---|
| 1 | T1211 cfapi + EICAR | 60 s | ~11.4 s | CFAPI registration + EICAR drop; sync-root provider connection can stall |
| 2 | T1006 VSS enum + oplock | 30 s | ~1.4 s | NT object enum is fast; FSCTL oplock returns synchronously |
| 3 | T1574 reparse + race | 60 s | ~5.3 s | FSCTL_SET/GET_REPARSE_POINT + NtCreateFile race loop is internally bounded |
| 4 | T1559.001 COM activation | 60 s | ~9 s | `CoCreateInstance` can hang under contention — watchdog is load-bearing |

## Exit Code Semantics

| Exit | Meaning |
|---|---|
| 101 | All 4 stages completed unimpeded — endpoint is UNPROTECTED against the RedSun primitive chain |
| 126 | At least one stage was blocked/flagged by EDR — endpoint PROTECTED |
| 105 | A stage binary was quarantined on extraction |
| 999 | Prerequisite failure (e.g., Cloud Files feature not installed) |

Stage 4 will short-circuit cleanly with success code 0 if `TieringEngineService` is in any state other than STOPPED — the pre-check guard fires before any COM activation. The state-query telemetry is the value of the stage in that case.
