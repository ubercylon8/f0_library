## Description

Simulates the complete credential harvesting chain of the AMOS (Atomic Stealer) and Banshee Stealer macOS infostealer families, which operate as Malware-as-a-Service (MaaS) platforms at $3,000/month. These stealers represent the industrialization of macOS credential theft, with Red Canary documenting a 400% increase in macOS threats from 2023 to 2024. The test covers 8 sequential phases replicating real-world stealer behavior: fake password dialog via osascript, credential validation via dscl, Keychain extraction via Chainbreaker, browser credential theft across 9+ browsers, cryptocurrency wallet enumeration, TCC database manipulation, XProtect-style XOR string encryption for evasion, and data staging/exfiltration via HTTP POST with hardware and worker identifiers.

## Technical Details

### Attack Flow

1. **Phase 1: osascript Credential Phishing** (T1059.002, T1056.002)
   - Creates a realistic AppleScript credential dialog artifact matching the exact pattern used by AMOS
   - Includes Banshee's 5-retry logic and dscl password validation
   - Documents the Cuckoo variant's pw.dat storage in .local-UUID hidden directory
   - Creates osascript command execution log

2. **Phase 2: dscl Credential Validation** (T1059.002)
   - Documents the `dscl /Local/Default -authonly <user> <password>` validation pattern
   - This is the universal validation step shared by AMOS, Banshee, and Cuckoo
   - Logs the validation flow and additional dscl reconnaissance commands

3. **Phase 3: Keychain Credential Dumping** (T1555.001)
   - Simulates `security list-keychains -d user` enumeration
   - Generates Chainbreaker-style extraction output with 10+ keychain entries
   - Includes Chrome Safe Storage key extraction (critical for browser credential decryption)
   - Targets login.keychain-db, System.keychain, and iCloud.keychain
   - Exports structured JSON of extracted entries

4. **Phase 4: Browser Credential Theft** (T1005)
   - Enumerates credential database paths across 9 browsers: Chrome, Firefox, Safari, Brave, Edge, Opera, Vivaldi, Chromium, Arc
   - Documents Chrome Safe Storage key extraction from Keychain
   - Simulates Safari cookie restoration (AMOS feature for expired cookie replay)
   - Includes Apple Notes (NoteStore.sqlite) extraction
   - Generates simulated credential extraction results with encryption indicators

5. **Phase 5: Cryptocurrency Wallet Targeting** (T1005)
   - Enumerates 8 cryptocurrency wallets: MetaMask, Coinbase, Exodus, Atomic, Electrum, Phantom, Trust Wallet, Bitwarden
   - Documents exact file paths used by real stealers
   - Reports data sizes, file counts, and keystore availability
   - Exports structured JSON wallet enumeration data

6. **Phase 6: TCC Database Reset** (Cuckoo Pattern)
   - Simulates `tccutil reset AppleEvents` command
   - Documents the purpose: clearing consent decisions to force re-prompting
   - Explains impact on Automation permissions

7. **Phase 7: XProtect XOR String Encryption** (T1027)
   - Implements the XOR encryption algorithm adopted by Banshee from Apple's XProtect
   - Encrypts 8 sample strings commonly targeted by AV signatures
   - Documents how this technique evaded VirusTotal for 2+ months
   - Exports XOR-encrypted samples with hex output

8. **Phase 8: Data Staging and Exfiltration** (T1560.001, T1041)
   - Creates simulated "out.zip" archive with ZIP magic bytes
   - Generates detailed archive manifest (67 files, ~48.5 MB)
   - Documents HTTP POST multipart/form-data exfiltration with hwid/wid/user fields
   - Creates stealer configuration JSON with build ID and target lists
   - Documents Banshee's XOR-encrypted C2 variant

- osascript process execution with `display dialog` and `with hidden answer`
