**MITRE ATT&CK**: T1562.001

## Description

This test evaluates EDR/AV capabilities against the SilentButDeadly network isolation technique. SilentButDeadly uses the Windows Filtering Platform (WFP) to surgically block EDR network communications without terminating security processes. This is a stealthier approach than traditional EDR killing techniques because security services continue running - they simply cannot communicate with their cloud infrastructure.

The technique is particularly dangerous because:
- No process termination alerts are generated
- EDR agents appear to be running normally
- Cloud-based threat intelligence is cut off
- Real-time telemetry upload is blocked
- Detection and response capabilities are degraded

## Technical Details

### Attack Flow

1. **Phase 0 - Initialization**
   - Dropper framework initialization
   - Logger setup with Schema v2.0 compliance

2. **Phase 1 - Prerequisites Check**
   - Verify administrator privileges (required for WFP)
   - Validate system environment

3. **Phase 2 - Binary Extraction**
   - Extract embedded SilentButDeadly.exe to c:\F0\
   - Wait for defensive reaction (3 seconds)
   - Check for file quarantine

4. **Phase 3 - WFP EDR Network Isolation**
   - Execute SilentButDeadly with full output capture
   - Tool enumerates EDR processes
   - Tool initializes WFP engine
   - Tool creates blocking filters per EDR process
   - Tool monitors for ~30 seconds
   - Tool cleans up filters on exit

### Key Indicators

- **File System**: SilentButDeadly.exe dropped to c:\F0\
- **Process**: SilentButDeadly.exe execution with elevated privileges
- **WFP**: Filter creation in FWPM_LAYER_ALE_AUTH_CONNECT and FWPM_LAYER_ALE_AUTH_RECV_ACCEPT layers
- **Network**: Sudden loss of EDR cloud connectivity

## Expected Results

### Unprotected System (Code 101)

When the attack succeeds:
- SilentButDeadly executes without interruption
- WFP filters are successfully created
- EDR processes are network-isolated
- Cloud connectivity is blocked for ~30 seconds
- Filters are automatically cleaned up on exit
- No process termination occurs

The system is vulnerable to this EDR evasion technique. Attackers could use this window to execute additional malicious activities while EDR telemetry is disrupted.

### Protected System (Enhanced Detection)

- **Code 105**: SilentButDeadly.exe quarantined on extraction
  - EDR file-based detection identified the attack tool
  - Binary was removed before execution could occur

- **Code 126**: Execution prevented or WFP operations blocked
  - Behavioral detection identified malicious WFP usage
  - EDR self-protection prevented filter application
  - Process was terminated during execution
