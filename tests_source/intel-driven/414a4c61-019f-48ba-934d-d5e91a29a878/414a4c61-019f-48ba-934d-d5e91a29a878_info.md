## Description

This test simulates the UNK_RobotDreams threat actor's multi-stage attack chain as documented in threat intelligence reports on the 2026 Middle East Conflict Cluster. UNK_RobotDreams is a sophisticated, suspected Pakistan-aligned threat actor that strategically targets India-based offices of Middle Eastern government organizations using custom, high-performance Rust malware and Microsoft's Azure Front Door CDN for domain fronting C2.

The test implements a 3-stage sequential killchain where each stage maps to distinct MITRE ATT&CK techniques. If any stage is blocked by EDR, the test halts and reports which specific technique was detected, providing technique-level detection precision for security operations teams.

## Technical Details

### Attack Flow

1. **Stage 1: Malicious PDF Lure Preparation (T1204.002)**
   - Creates PDF file `Gulf_Security_Alert_MEA_2026.pdf` themed as Indian Ministry of External Affairs advisory
   - PDF includes OpenAction URI referencing Azure Front Door URL for executable download
   - Creates simulated executable loader `AdobeAcrobatUpdate.exe` (benign batch file) in ARTIFACT_DIR
   - Creates URL shortcut `Read_Gulf_Security_Alert.url` linking to the loader
   - Tests whether EDR detects PDF with suspicious OpenAction + executable in same directory

2. **Stage 2: PowerShell Staging and Payload Retrieval (T1059.001 + T1105)**
   - Creates simulated Rust backdoor `agent.exe` (benign) at ARTIFACT_DIR
   - Executes PowerShell with `-WindowStyle Hidden -NoProfile -ExecutionPolicy Bypass`
   - PowerShell runs Invoke-WebRequest targeting localhost:65535 (safe URL, models real download cradle pattern)
   - Executes second PowerShell command simulating Start-Process on the staged payload
   - Tests whether AMSI/EDR blocks the hidden PowerShell download and execution chain

3. **Stage 3: Azure Front Door C2 Communication (T1071.001 + T1573.001 + T1036.005)**
   - Collects system metadata: hostname, username, computer name, OS version, IP address
   - Encrypts metadata with AES-256-GCM symmetric encryption (T1573.001)
   - Stages encrypted beacon data to `c:\F0\c2_staging\beacon.dat`
   - Attempts HTTPS POST to localhost:443 with Host header `f0rt1ka-test.azurefd.net` (domain fronting)
   - Attempts HTTPS POST to localhost:8443 with Host header `f0rt1ka-cdn.azureedge.net`
   - Uses Azure Front Door specific headers (X-Azure-Ref, X-FD-HealthProbe) to model CDN abuse

### Key Indicators

- PDF with OpenAction URI pointing to Azure Front Door domain
- Executable named `AdobeAcrobatUpdate.exe` in user profile directory
- PowerShell execution with `-WindowStyle Hidden` and `Invoke-WebRequest`
- PowerShell `-ExecutionPolicy Bypass` flag
- File named `agent.exe` written to user profile
- HTTPS POST requests with Azure Front Door Host headers from non-browser processes
- AES-encrypted beacon data file in staging directory
- Azure-specific HTTP headers (X-Azure-Ref, X-FD-HealthProbe) from non-Azure processes

## Detection Opportunities

1. **PDF Lure Detection** -- PDF files with OpenAction URI pointing to executable downloads, especially referencing CDN endpoints
2. **PowerShell Download Cradle** -- PowerShell.exe with hidden window flag executing Invoke-WebRequest to download executables
3. **Domain Fronting via CDN** -- HTTPS requests where the Host header references Azure Front Door/CDN domains but the actual destination is unusual
4. **Encrypted Beacon Data** -- AES-encrypted files staged in temporary or logging directories with base64 encoding
5. **Combined Behavioral Correlation** -- PDF creation + PowerShell execution + HTTPS C2 on same host within short timeframe

## Expected Results

### Unprotected System (Code 101)
- All 3 stages complete successfully
- PDF lure and executable loader created without quarantine
- PowerShell download cradle executed with AMSI bypass
- C2 HTTPS POST with domain fronting headers attempted
- Encrypted beacon data staged to disk
- **Impact**: Full UNK_RobotDreams backdoor operational on endpoint

### Protected System (Enhanced Detection)
- **Code 105**: Stage binary quarantined before execution (AV signature detection)
- **Code 126**: Stage blocked during execution (EDR behavioral detection)
  - Stage 1 blocked: PDF lure or executable loader quarantined
  - Stage 2 blocked: PowerShell download cradle blocked by AMSI or EDR
  - Stage 3 blocked: C2 communication or beacon staging prevented
