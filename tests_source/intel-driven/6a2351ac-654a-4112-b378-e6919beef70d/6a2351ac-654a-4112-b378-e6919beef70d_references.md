## Simulation Build Notes

- **Architecture decision**: multi-stage — three distinct detection opportunities (registry recon, file-lock primitive, service-notification subscription) warrant independent stage binaries for detection-engineering precision.
- **Safety deviation from source**: stage 2's `LockFile`/`LockFileEx` primitive targets a self-created file under `ARTIFACT_DIR`, not a real Defender file. This reproduces the behavioral signal without risking persistent Defender DoS between test runs.
- **Service-notification callback**: the real UnDefend calls `SleepEx(INFINITE, TRUE)` to wait for the callback. This simulation deliberately returns without an alertable wait so the callback cannot fire.
