## Description

This multi-stage test simulates the complete attack chain of APT42 (also tracked as Magic Hound, Educated Manticore, and UNC788), an Iranian state-sponsored threat group that targets organizations in the financial, government, and defense sectors. The test specifically replicates the TAMECAT fileless backdoor deployment pattern, which uses LNK files delivered via WebDAV to initiate a VBScript-based downloader, followed by in-memory PowerShell execution, dual persistence mechanisms, browser credential theft, and multi-channel data exfiltration via Telegram API.

The financial relevance of this test is significant: APT42 specifically targets banking applications, trading platforms, payment portals, and treasury management systems that are accessed through web browsers. The credential theft stage simulates the exact techniques used to harvest browser-stored credentials for these financial platforms.

## Technical Details

### Attack Flow

1. **Stage 1 - Initial Access (T1204.002, T1059.005)**
   - Creates realistic Windows .lnk shortcut file (binary format) disguised as PDF
   - LNK targets cscript.exe with VBScript downloader argument
   - VBScript enumerates AV products via WMI SecurityCenter2 namespace
   - Checks Windows Defender status via root\Microsoft\Windows\Defender namespace
   - Gathers system information (hostname, domain, username, OS version)

2. **Stage 2 - TAMECAT Backdoor (T1059.001)**
   - Creates TAMECAT-style PowerShell payload encoded in UTF-16LE base64
   - Executes via conhost.exe --headless -> powershell.exe -EncodedCommand chain
   - Performs environment fingerprinting (AMSI detection, Defender process enumeration)
   - Creates beacon marker file simulating C2 check-in
   - Tests secondary IEX (Invoke-Expression) in-memory execution pattern

3. **Stage 3 - Dual Persistence (T1547.001, T1037.001)**
   - Creates Registry Run key: HKCU\...\Run\Renovation -> cscript.exe payload
   - Creates UserInitMprLogonScript: HKCU\Environment\UserInitMprLogonScript
   - Verifies both values were actually written (detects silent EDR removal)
   - Automatic cleanup of both persistence mechanisms after test

4. **Stage 4 - Browser Credential Theft (T1555.003)**
   - Attempts Edge remote debugging port activation (port 9222)
   - Attempts to copy Chrome Login Data SQLite database
   - Attempts to copy Edge Login Data SQLite database
   - Simulates Runs.dll data chunking (4KB chunks for staged exfiltration)
   - All credential copies immediately deleted after size measurement

5. **Stage 5 - Multi-Channel Exfiltration (T1102)**
   - Tests DNS resolution and HTTPS connectivity to api.telegram.org
   - Sends benign POST to Telegram sendMessage API (invalid token, tests connection pattern)
   - Creates FTP upload staging artifacts (simulated data, cleaned up)
   - Tests HTTPS POST exfiltration to external endpoint
   - Evaluates which channels are blocked by network security controls

### Key Indicators
- LNK file creation: `c:\F0\Important_Document.pdf.lnk`
- VBScript file: `c:\F0\update_check.vbs`
- WMI queries to `root\SecurityCenter2` and `root\Microsoft\Windows\Defender`
- PowerShell `-EncodedCommand` execution from conhost.exe context
- Registry value `Renovation` in `HKCU\...\CurrentVersion\Run`
- Registry value `UserInitMprLogonScript` in `HKCU\Environment`
- File access to Chrome/Edge `Login Data` SQLite databases
- Edge launched with `--remote-debugging-port=9222`
- Outbound HTTPS to `api.telegram.org`
- HTTPS POST to external data collection endpoints

## Detection Opportunities

1. **File System Activity**
   - LNK file creation with cscript.exe target in WebDAV-style path
   - VBScript file creation with WMI enumeration patterns
   - Browser credential database access/copy attempts
   - FTP staging directory creation

2. **Process Behavior**
   - cscript.exe executing VBScript from c:\F0
   - conhost.exe --headless spawning powershell.exe
   - PowerShell -EncodedCommand with large base64 payload
   - Edge launched with --remote-debugging-port flag

3. **Registry Modifications**
   - HKCU Run key creation with suspicious value name "Renovation"
   - UserInitMprLogonScript creation (uncommon persistence technique)
   - Both modifications targeting script execution paths

4. **Network Communications**
   - DNS resolution of api.telegram.org from workstation
   - HTTPS POST to api.telegram.org/bot* endpoints
   - FTP port 21 connectivity attempts
   - Large HTTPS POST payloads to external services

5. **Script/AMSI Detection**
   - PowerShell Script Block Logging content
   - AMSI scan events for encoded PowerShell
   - VBScript WMI query patterns (SecurityCenter2, MSFT_MpComputerStatus)
