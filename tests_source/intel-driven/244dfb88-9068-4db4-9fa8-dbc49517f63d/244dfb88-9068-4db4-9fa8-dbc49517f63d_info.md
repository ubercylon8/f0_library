## Description

This test simulates the complete DPRK/BlueNoroff attack chain targeting macOS endpoints in the financial and cryptocurrency sectors. BlueNoroff is a subgroup of the Lazarus Group, specifically tasked with generating revenue for the DPRK regime through attacks on financial institutions, cryptocurrency exchanges, and blockchain companies.

The simulation models techniques documented across five BlueNoroff campaigns:
- **RustBucket** (2023-2024): Multi-stage dropper chain (AppleScript -> Objective-C -> Rust) with LaunchAgent persistence
- **Hidden Risk** (November 2024, SentinelLabs): Abuses `~/.zshenv` for stealth persistence that bypasses macOS Ventura's Login Items notification, uses notarized malware signed with hijacked Apple Developer ID
- **KANDYKORN** (2023): Full RAT targeting blockchain engineers using `linkpc.net` dynamic DNS domains
- **TodoSwift** (2024): Swift/SwiftUI dropper leveraging Google Drive URLs for payload delivery and exfiltration
- **BeaverTail** (2024): Qt framework-based credential stealer delivering InvisibleFerret Python backdoor, targeting crypto developers

## Technical Details

### Attack Flow

1. **Stage 1: Gatekeeper Bypass & Payload Delivery** (T1553.001)
   - Creates simulated curl-based dropper script (InternalPDFViewer.sh)
   - Writes fake code signing metadata with hijacked Apple Developer ID ("Avantis Regtech Private Limited")
   - Creates quarantine attribute removal script (xattr -d com.apple.quarantine)
   - Drops fake macOS application manifest (CryptoExchangePro_Info.plist)
   - Artifacts placed in `c:\Users\fortika-test` for EDR detection

2. **Stage 2: LaunchAgent Persistence** (T1543.004)
   - Creates `com.apple.systemupdate.plist` in simulated ~/Library/LaunchAgents/ (RustBucket)
   - Creates `com.avatar.update.wake.plist` (BeaverTail/InvisibleFerret)
   - Modifies `.zshenv` with Hidden Risk payload (bypasses Login Items notification)
   - Creates `com.apple.security.updateagent.plist` in simulated LaunchDaemons (root persistence)

3. **Stage 3: Credential Harvesting** (T1059.002, T1555.001, T1056.002)
   - Creates osascript fake password dialog simulation
   - Creates dscl credential validation script
   - Creates Keychain dump simulation with extracted credentials
   - Creates browser credential CSV (Chrome Login Data pattern)
   - Stages crypto wallet data: MetaMask vault, Exodus wallet, Coinbase Wallet
   - Creates TCC manipulation script simulation

4. **Stage 4: C2 Communication** (T1071.001, T1573.002, T1071.004)
   - Creates Sliver mTLS beacon configuration (port 8888, beacon.linkpc.net)
   - Creates HTTPS fallback C2 config with Hidden Risk, RustBucket, KANDYKORN endpoints
   - Creates DNS tunnel configuration with base64-encoded subdomain pattern
   - Creates TodoSwift Google Drive staging configuration
   - Performs DNS resolution against linkpc.net domains (passive, safe)
   - Generates system beacon payload with hwid, wid, user identifiers

5. **Stage 5: Data Exfiltration** (T1041, T1567.002, T1560.001)
   - Aggregates all staged credential and wallet data
   - Compresses to zip archive using AMOS "out.zip" naming pattern
   - Simulates AWS S3 upload with hardcoded credentials (NotLockBit pattern)
   - Simulates Google Drive exfiltration (TodoSwift pattern)
   - Simulates HTTP POST exfiltration with campaign metadata identifiers
   - Creates detailed exfiltration summary with channel accounting

## Detection Opportunities

1. **Plist File Creation** (File System)
   - Monitor for plist files with RunAtLoad=true and KeepAlive in user LaunchAgents directory
   - Alert on plist labels masquerading as Apple services (com.apple.*)
   - Detect plist creation by non-Apple processes

2. **.zshenv Modification** (File System)
   - Monitor changes to ~/.zshenv (should rarely be modified by applications)
   - Detect curl/wget commands embedded in shell configuration files
   - Alert on environment variable exports containing URLs or C2 indicators

3. **osascript Abuse** (Process Behavior)
   - Monitor osascript execution with "display dialog" and "with hidden answer"
   - Detect password prompt dialogs from non-system processes
   - Alert on dscl -authonly credential validation attempts

4. **Keychain Access** (Process Behavior)
   - Monitor security command-line tool usage (dump-keychain, find-generic-password)
   - Detect Chrome Safe Storage key extraction
   - Alert on bulk Keychain enumeration (list-keychains followed by dump)

5. **Crypto Wallet Data Access** (File System)
   - Monitor access to browser extension storage directories (MetaMask, Coinbase Wallet)
   - Detect reads of Exodus wallet data files
   - Alert on staging of multiple wallet-related files

6. **C2 Network Indicators** (Network)
   - Block/alert on linkpc.net and related dynamic DNS domains
   - Monitor mTLS connections on non-standard ports (8888)
   - Detect DNS tunneling patterns (long subdomain labels, high query rates)
   - Alert on Google Drive API uploads from non-browser processes

7. **Data Exfiltration** (File System / Network)
   - Monitor zip archive creation in staging directories
   - Detect archives containing credential-related filenames
   - Alert on bulk file compression followed by HTTP POST or cloud upload
   - Monitor for AWS SDK activity from unexpected processes
