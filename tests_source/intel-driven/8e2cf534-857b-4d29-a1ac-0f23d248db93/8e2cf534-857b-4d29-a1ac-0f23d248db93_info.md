## Description

This test simulates the TA453 NICECURL VBScript backdoor as documented in threat intelligence on Iran-aligned IRGC-IO operations. NICECURL represents TA453's Cluster D shift toward Living off the Land (LotL) techniques, using legitimate Windows utilities (wscript.exe, curl.exe, wmic.exe) to blend with normal system activity.

The test implements a 3-stage sequential killchain where each stage maps to distinct MITRE ATT&CK techniques. If any stage is blocked by EDR, the test halts and reports which specific technique was detected, providing technique-level detection precision for security operations teams.

## Technical Details

### Attack Flow

1. **Stage 1: NICECURL Delivery via Malicious LNK (T1204.002 + T1059.005 + T1036.004)**
   - Creates VBScript simulating NICECURL backdoor initialization (benign marker file)
   - Creates minimal Windows .lnk file named `onedrive-form.pdf.lnk` (double extension masquerading)
   - LNK targets `wscript.exe //Nologo //B` with the VBScript as argument
   - Executes wscript.exe to trigger the LNK -> wscript.exe -> VBScript chain
   - Verifies VBScript execution via marker file at `c:\F0\nicecurl_vbs_executed.txt`

2. **Stage 2: WMI Security Software Discovery (T1047 + T1518.001)**
   - Executes `wmic.exe /Namespace:\\root\SecurityCenter2 Path AntiVirusProduct Get displayName,productState`
   - Creates and executes VBScript that uses `GetObject("winmgmts:").ExecQuery("Select * from AntiVirusProduct")`
   - Creates persistent victim identifier at `%LOCALAPPDATA%\config.txt` matching NICECURL format
   - Logs discovered AV products

3. **Stage 3: curl.exe HTTPS C2 Communication (T1071.001 + T1105)**
   - Locates `curl.exe` in System32 (available on Windows 10 1803+)
   - Constructs base64-encoded beacon payload with victim ID, hostname, username
   - Uses curl.exe for HTTPS POST to `accurate-sprout-porpoise.glitch.me` (Glitch-themed C2)
   - Custom headers: `User-Agent: NICECURL/2.0`, `X-Request-ID: <victim-id>`
   - Simulates tool transfer (T1105) via curl download attempt
   - DNS resolution fails safely (non-existent domain)

### Key Indicators

- LNK file with double extension (`.pdf.lnk`) targeting wscript.exe
- wscript.exe executing VBScript from user-writable directories
- WMI queries to `root\SecurityCenter2\AntiVirusProduct` namespace
- `config.txt` created in %LOCALAPPDATA% containing victim identifier
- curl.exe HTTPS POST to `*.glitch.me` domains from non-interactive context
- Base64-encoded beacon data in curl POST body
- Custom User-Agent `NICECURL/2.0` from curl.exe

## Detection Opportunities

1. **LNK Delivery Chain** -- LNK files with wscript.exe targets and double-extension masquerading (.pdf.lnk)
2. **VBScript Execution** -- wscript.exe/cscript.exe spawning from explorer.exe or running VBS from non-standard paths
3. **WMI Discovery** -- wmic.exe or VBScript querying SecurityCenter2 AntiVirusProduct for security software enumeration
4. **curl.exe C2** -- curl.exe making HTTPS POST to suspicious domains, especially Glitch.me, from non-interactive context
5. **Combined Behavioral Correlation** -- Multiple indicators on same host within short timeframe
