## Overview

This test simulates the behavior of Gunra Ransomware, a recently observed ransomware strain that employs advanced encryption methods and double-extortion tactics. Gunra targets Windows systems globally across various industries including real estate, pharmaceuticals, and manufacturing. The test replicates key malicious behaviors to validate endpoint detection and response (EDR) capabilities.

## Test Behavior Summary

1. **Anti-Debugging Check**: Simulates IsDebuggerPresent API call used by Gunra to detect analysis environments
2. **System Information Discovery**: Gathers hostname, current user, and OS information
3. **Test Environment Setup**: Creates test directory at `c:\F0\94b248c0-a104-48c3-b4a5-3d45028c407d` with sample files
4. **Shadow Copy Deletion Simulation**: Checks access to Volume Shadow Copy service (safe mode - no actual deletion)
5. **File Enumeration**: Scans test directory for files to process
6. **File Encryption Simulation**: Renames files with .ENCRT extension
7. **Ransom Note Deployment**: Drops "R3ADM3.txt" ransom note in test directory
8. **Verification**: Confirms encrypted files and ransom note exist

## Safety Features

- **No actual encryption**: Files are renamed only, no cryptographic operations
- **Isolated test directory**: All operations confined to `c:\F0\94b248c0-a104-48c3-b4a5-3d45028c407d`
- **Safe shadow copy mode**: Only lists shadows, does not delete
- **Cleanup script provided**: `Cleanup-GunraTest.ps1` for artifact removal
- **Clear test indicators**: Ransom note explicitly states this is a security test
