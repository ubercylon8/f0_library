## Overview
This test replicates the CyberEye RAT's sophisticated technique for disabling Windows Defender through PowerShell-based registry modifications. The test evaluates endpoint security solutions' ability to detect and prevent malicious PowerShell scripts that attempt to neutralize Windows Defender's protection mechanisms.

## Technique Details
- **MITRE ATT&CK**: T1562.001 - Impair Defenses: Disable or Modify Tools
- **Tactic**: Defense Evasion
- **Platform**: Windows
- **Permissions Required**: Administrator
- **Category**: Defense Evasion, PowerShell Abuse, Registry Manipulation

## Test Description
The test simulates the exact attack chain used by CyberEye RAT:

1. **Script Deployment**: Drops a PowerShell script (CyberEye-TTPs.ps1) to c:\F0
2. **Execution Policy Bypass**: Launches PowerShell with `-ExecutionPolicy Bypass` parameter
3. **Administrative Check**: Verifies administrative privileges are available
4. **Registry Manipulation**: Performs systematic registry modifications:
   - Disables Tamper Protection: `HKLM:\SOFTWARE\Microsoft\Windows Defender\Features\TamperProtection = 0`
   - Disables Anti-Spyware: `HKLM:\SOFTWARE\Policies\Microsoft\Windows Defender\DisableAntiSpyware = 1`
   - Disables Real-Time Protection components in `HKLM:\SOFTWARE\Policies\Microsoft\Windows Defender\Real-Time Protection`:
     - `DisableBehaviorMonitoring = 1`
     - `DisableOnAccessProtection = 1`
     - `DisableScanOnRealtimeEnable = 1`
5. **Verification**: Queries current Defender settings to confirm modifications

## Expected Behaviors
- **Detection**: Security solutions should identify:
  - PowerShell execution with suspicious parameters
  - Registry modifications in Windows Defender paths
  - Attempts to disable security services
  - Execution policy bypass attempts
  - Pattern of registry keys being modified sequentially

- **Prevention**: Advanced endpoint protection should block:
  - PowerShell script execution with malicious content
  - Registry write operations to security-related keys
  - Administrative privilege abuse for security bypass
  - Process execution attempting to modify system security

- **Alerting**: Expected security alerts:
  - PowerShell script execution with bypass parameters
  - Registry modification attempts in Windows Defender keys
  - Potential malware behavior - security software tampering
  - Administrative privilege escalation for malicious purposes

## Success Metrics
- **101 (Unprotected)**: PowerShell script executes successfully AND registry modifications complete (3+ successful registry changes), Windows Defender components disabled
- **105 (File Quarantined)**: PowerShell script is quarantined upon extraction to c:\F0
- **126 (Execution Prevented)**: Registry modifications blocked with "Requested registry access is not allowed" or similar permission denied errors - indicates proper system protection
