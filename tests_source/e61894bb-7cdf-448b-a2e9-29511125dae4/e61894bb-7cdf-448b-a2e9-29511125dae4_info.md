**Key Findings**: Qilin ransomware group has evolved their techniques to use Linux binaries directly on Windows systems, leveraging WSL and cross-platform execution to evade traditional Windows-focused EDR solutions.

## Attack Techniques Simulated

### 1. WSL Abuse for Cross-Platform Execution
- **MITRE ID**: T1202 (Indirect Command Execution)
- **Description**: Uses Windows Subsystem for Linux to execute Linux binaries that may evade Windows EDR monitoring
- **Implementation**: Verifies WSL installation, installs if needed, executes Linux payload

### 2. Linux Binary Evasion
- **MITRE ID**: T1140 (Deobfuscate/Decode Files or Information)
- **Description**: Executes reconnaissance and attack tools as Linux binaries to bypass Windows-based detection
- **Implementation**: Shell script performing system discovery via Linux commands

### 3. Credential Harvesting
- **MITRE ID**: T1003 (OS Credential Dumping)
- **Description**: Targets specific applications like Veeam Backup and SQL databases for credential theft
- **Implementation**: PowerShell simulation of credential extraction from multiple sources

### 4. BYOVD (Bring Your Own Vulnerable Driver)
- **MITRE ID**: T1574.002 (Hijack Execution Flow: DLL Side-Loading)
- **Description**: Loads signed but vulnerable drivers to gain kernel-level access
- **Implementation**: Attempts to load rwdrv.sys vulnerable driver

### 5. Cross-Platform Lateral Movement
- **MITRE ID**: T1046 (Network Service Scanning)
- **Description**: Uses Linux networking tools for reconnaissance and lateral movement
- **Implementation**: Network scanning and SSH connection attempts via WSL

## Test Phases

1. **WSL Verification & Installation** (Required per user request)
   - Checks WSL status
   - Installs WSL if not present
   - Verifies functionality

2. **Linux Payload Execution**
   - Drops and executes Linux script via WSL
   - Performs system reconnaissance
   - Creates attack indicators

3. **Credential Harvesting**
   - Simulates Veeam credential theft
   - SQL database credential extraction
   - Domain and browser credential search

4. **BYOVD Attack**
   - Loads vulnerable driver
   - Simulates kernel-level access
   - Demonstrates privilege escalation

5. **Lateral Movement**
   - Network reconnaissance via Linux tools
   - Cross-platform connection attempts
   - Simulates environment expansion

## Expected EDR Reactions

### Protective Systems Should:
- Block or monitor WSL usage for security purposes
- Detect Linux binary execution in Windows context
- Prevent credential harvesting attempts
- Block vulnerable driver loading
- Alert on cross-platform attack patterns

### Vulnerable Systems Will:
- Allow unrestricted WSL usage
- Miss Linux binary execution monitoring
- Fail to protect stored credentials
- Allow vulnerable driver loading
- Lack cross-platform attack visibility

## Risk Assessment
This test evaluates protection against sophisticated, multi-platform evasion techniques that represent the evolution of modern ransomware attacks. Organizations without cross-platform security visibility are particularly vulnerable.

## Indicators of Compromise (IOCs)
- WSL installation activity
- Linux binary execution via WSL
- Credential access attempts
- Vulnerable driver loading (rwdrv.sys)
- Cross-platform network reconnaissance
- File system artifacts in /tmp/ (Linux) and C:\F0\ (Windows)
