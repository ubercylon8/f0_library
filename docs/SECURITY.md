# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of F0RT1KA seriously. If you have discovered a security vulnerability in our project, we appreciate your help in disclosing it to us in a responsible manner.

### Reporting Process

1. **DO NOT** create a public GitHub issue for security vulnerabilities
2. Email your findings to james@fortika.io with:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- Acknowledgment of your report within 48 hours
- Regular updates on our progress
- Credit for responsible disclosure (unless you prefer to remain anonymous)

### Security Considerations for F0RT1KA

Given the nature of this security testing framework:

1. **Authorized Use Only**: This framework should only be used on systems you own or have explicit written permission to test
2. **Isolation**: Run tests in isolated environments when possible
3. **Monitoring**: Be aware that tests may trigger security alerts
4. **Cleanup**: Ensure all test artifacts are properly removed after testing

### Best Practices

When using F0RT1KA:
- Always verify you have proper authorization
- Use dedicated testing environments
- Monitor system changes during tests
- Document all testing activities
- Follow your organization's security policies

### Disclosure Policy

- We will work to verify and reproduce reported vulnerabilities
- We will develop and test fixes privately
- We will coordinate disclosure with the reporter
- We will publicly disclose the vulnerability after a fix is available

Thank you for helping keep F0RT1KA and its users safe!
